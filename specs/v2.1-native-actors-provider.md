# ScalaScript 2.1 core-free native Actors provider

Status: TI-8.2d3j implementation contract (2026-07-12)

## Goal

The standard ScalaScript 2.1 launcher must execute the canonical local actor
and typed named-loopback examples on native VM/direct ASM without
`PluginBridge`, the v1 actor scheduler/interpreter, Scalameta, or a compiler.

The exact public outputs are:

```text
pong: one
pong: two
pong: three
after timeout: None
before timeout: Some(got delivered)
done
```

and:

```text
true
true
local ref
spawnRemote: pong
```

## Baseline

Installed `origin/main` through `1bfb39884` exits 1 on both examples and both
standard engines at `unbound global: runActors`, with no stdout. The explicit
compatibility tier prints the outputs above. Runtime taxonomy contains two
blocking `standard-provider` rows.

## Ownership and interface

A required provider at `v2/runtime/std/actors-plugin` owns:

- `runActors`, `spawn`, `receive`, `self`, `exit`, and the `actor.send`
  intrinsic used by `pid ! message`;
- the `@timeout` cell used by self-hosted named-argument lowering for
  `receive(timeout = ms) { handler }`;
- `startNode`, `registerBehavior`, `spawnRemote`, `globalWhereis`, and the
  typed local `ActorRef` method object (`address`, `isLocal`, `tryLocal`,
  `tell`, `publishAs`).

No missing operation is silently replaced by Unit or a compatibility value.

## Mailbox and lifecycle contract

- Each actor has one unbounded FIFO blocking mailbox and one JDK 21 virtual
  thread. Sends preserve enqueue order per sender and drop after the target is
  dead.
- `self()` is dynamically scoped to the current actor thread. Calling it
  outside `runActors`/`spawn` is an explicit error on the standard provider.
- Plain `receive(handler)` blocks for the next message and invokes the handler
  exactly once. Timed receive returns `Some(handler(message))` or `None` after
  the non-negative deadline. Negative timeout means an untimed receive.
- `exit(pid, reason)` marks the target dead and interrupts its thread.
  Subsequent sends are ignored. Exit is idempotent.
- `runActors(body)` creates a root actor and waits until its scope is quiescent:
  every actor is dead or blocked indefinitely on an empty mailbox. Timed waits
  are not quiescent. A delivery makes a blocked actor busy before enqueue, so
  quiescence cannot race ahead of message handling.
- Root and child failures are retained by the run scope and rethrown by
  `runActors` with actor context; they are never silently swallowed.

## Typed named loopback

`startNode(id)` selects the process-local node identity.
`registerBehavior(name, fn)` rejects duplicate replacement only by explicit
last-registration semantics and stores the closure in the provider registry.
`spawnRemote(node, name, arg)` remains a deliberate same-process loopback for
this provider: it starts the named behavior as an ordinary local actor but
retains `Some(node)` in the typed reference. Missing behaviors fail explicitly.

`publishAs(name)` and `globalWhereis(name)` share a process-local concurrent
registry. `tell` uses the same mailbox send contract; `tryLocal` exposes the
local mailbox only as an opaque value.

## Isolation

- The provider depends only on plugin SPI/v2 core, Scala runtime, and JDK.
- VM/direct ASM/build-jvm use the same provider and portable messages.
- Standard/slim distributions and deterministic artifacts require the provider
  jar and ServiceLoader entry.
- Scala 3 seed/image files remain unchanged.

## Out of scope

- Network transports, discovery, cluster membership/election, durable mailbox,
  supervision trees, links/monitors, and timer APIs. Those remain explicit
  advanced/distributed provider work and never fall back transparently.
