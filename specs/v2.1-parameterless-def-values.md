# ScalaScript 2.1 parameterless def values

## Overview

The self-hosted frontend must distinguish a parameterless method declaration
from the value returned when that method is referenced. A declaration such as
`def stage: Input => Output = input => ...` is a nullary computation whose
ordinary bare reference evaluates the computation and yields the function. It
must not expose the internal zero-arity closure as though it were the declared
function value.

## Interface

```scalascript
def stage: Int => Int = value => value + 1
val pipeline = source.andThen(stage)
println(pipeline(4))
```

The source-level reference `stage` lowers as an explicit zero-argument
application of its declared global. This is source auto-application, not a VM
arity exception or a runtime reflection rule.

The rule applies only to declarations written without a parameter clause.
`def stage(): Int => Int` retains its explicit-call contract, and declarations
with one or more parameter clauses remain first-class closures when referenced.

## Behavior

- [x] A bare reference to a parameterless def evaluates its nullary body and
      returns the declared value.
- [x] The reference works as a direct argument, through a local alias, and
      across a Markdown import boundary on VM and direct ASM.
- [x] The nullary computation is evaluated once per reference occurrence; the
      returned closure may be called repeatedly without re-running its factory.
- [x] An explicit-empty-parens def is not auto-applied, and higher-arity def
      references retain existing first-class-function behavior.
- [x] Selected/static owner dispatch and exact named-given/summon resolution do
      not acquire extra calls.
- [x] `dsl-mini-language.ssc` prints the exact 13-line explicit compatibility
      output on VM, direct ASM, and `build-jvm`.
- [x] Full parity retires only the mini-language row with zero mismatch or
      one-sided errors; release and fresh `v2-*` conformance gates pass.

## Design

The lowerer already records declaration arity. Add a durable set of globals
whose source declarations have no parameter clause and whose bodies therefore
lower as `lam 0`. During expression resolution, rewrite a bare `var(name)` in a
value position to `app(var(name), Nil)` only when `name` belongs to that set.
The declaration's own recursive body and the callee position of an already
explicit application must not be rewritten.

The distinction must be captured in the parsed/lowered module, not inferred
from the compiled `ClosV.arity`: runtime arity alone cannot distinguish
`def f: T` from `def f(): T`, and a runtime auto-retry would evaluate arbitrary
closures after a genuine caller error.

## Decisions

- **Frontend-owned auto-application** — chosen because parameter-clause shape
  is source semantics and is erased by CoreIR. Rejected: runtime “call arity 0,
  then retry” (ambiguous and can duplicate effects).
- **Exact declaration table** — chosen over name suffixes or result-type
  inspection; types are erased and imported names must behave identically.
- **No DSL special case** — `typeCheckPass` is the first public reproducer, not
  a privileged name or `Pass`-specific operation.

## Out of scope

- Scala eta-expansion beyond the existing first-class def behavior;
- implicit parameter insertion, by-name parameters, overload resolution, or
  changing explicit `()` call syntax;
- repairing the independent derives/Mirror, direct-do, optics, or SQL blockers.

## Results

`12d3d9cf2` preserves the omitted-parameter-clause distinction through a
transparent self-hosted AST marker and records the exact declarations during
lowering. A bare reference emits one direct zero-argument CoreIR application;
an already explicit callee is resolved without adding a second call. The
two-file imported fixture proves aliases, direct arguments, per-reference
effects, explicit `()`, and higher-arity function values with identical
nine-line VM/ASM output.

The same slice reconciles the independently exposed tuple-callback boundary,
so `dsl-mini-language.ssc` now prints all 13 compatibility lines on VM, direct
ASM, and deterministic `build-jvm`. The exhaustive release gate reports 48
identical / 18 both-fail / 129 skipped, zero mismatch or one-sided errors, five
blocking rows, and fresh `v2-*` conformance 11/11.
