# v2.2 P6.6 ‚Äî literal self-compilation fixpoint (no quine)

Status: **in progress.** Goal: a compiler that compiles **its own source**, reaching a stage1 == stage2
fixed point. This closes the literal end of the P6.5 self-host arc.

## The correction: there is no quine

An earlier note claimed a "quine problem" ‚Äî that to feed a compiler its own source you must embed the
source as a string literal in itself. **That is wrong.** The compiler reads its source **from a file**,
exactly the way the trusted reference runner `v2/bin/ssc1-run.ssc0` reads a `.ssc` file:

```
def main = () =>
  match #io.args() {
    case Cons(path, rest) =>
      let src = #utf8->str(#io.readFile(path)) in
      #io.print(compile(src))
    case Nil => #io.eprint("usage: <file>\n")
  }
```

The runtime primitives `#io.args`, `#io.readFile`, `#utf8->str`, `#io.print` all exist (`Runtime.scala`;
`ssc1-run.ssc0` uses them). A minimal ssc0 driver reading a file and printing its content has been verified
end to end.

## The one real subtlety (the driver, not a quine)

`#`-primitives are **ssc0**, not the Scala **subset** the self-host compiler is written in
(`def main(): Int = #i.add(3, 4)` compiled through `ssc1-front` yields `(global _err)` ‚Äî the subset front
does not know `#`-prims). So the **file reading is done by an ssc0 driver** that wraps the compiler's pure
`compile: String ‚Üí String` function; the compiler stays a pure function, the driver does the I/O. This is
exactly how `ssc1-run.ssc0` is structured (a driver around `lowerProg`).

## The fixed point

Let **C** be a compiler for a language **F**, written **in F**, emitting **executable Core IR**. Let
`driver(defs)` wrap a set of compiled `defs` (which include `compile`) in the file-reading `main` above.

1. **Bootstrap.** The spike (front + `ssc1-lower`) compiles `C`'s source (F ‚äÜ subset) ‚Üí `C0 = driver(spike(C))`.
2. **Stage 1.** Run `C0` with `C`'s own source file ‚Üí `stage1 = C0.compile(C_src)` ‚Äî C's definitions, compiled
   by the spike-built C.
3. **Stage 2.** Run `C1 = driver(stage1)` with `C`'s own source file ‚Üí `stage2 = C1.compile(C_src)`.
4. **Fixed point.** `stage1 == stage2` (byte-identical). This holds iff `C0.compile` correctly compiles F
   (including C's own source): a correct `C0.compile(C_src)` is a `compile` function equivalent to
   `C0.compile`, so `C1.compile(C_src) == C0.compile(C_src) == stage1`. `scalac`/the spike stays the outer
   oracle for the bootstrap.

Nothing here embeds C in itself. C's source is a **file** the driver reads at runtime.

## The language F

F must be rich enough to write C in, and F ‚äÜ subset (so the spike bootstraps C). From the P6.5 work, F =
**multi-param functions, recursion, `if`/`then`/`else`, pattern `match` (`Nil`/`Cons`/tuple/**var**),
`Cons`/`Nil`/tuple construction, strings + `charAt`/`substring`/`length`/`==`/`+`(concat), integers,
polymorphic arithmetic, multi-arg calls**, and **local bindings**.

Two design choices make C's *own source* valid F **without new escape/var-pattern machinery** (all
mechanisms below are probed + proven runnable):

- **Local bindings via helper functions, not `let ‚Ä¶ in`.** `let ‚Ä¶ in` is not Scala subset syntax (bootstrap
  can't parse it). Instead, lift each `val x = e; body` into a helper `def h(‚Ä¶, x) = body` called as
  `h(‚Ä¶, e)` ‚Äî the bound value becomes a **parameter**. Uses only multi-param functions (already in F); no
  var-patterns, no offside blocks.
- **No string escapes ‚Äî a quote-free emission design.** Two tricks together mean C's source contains **no**
  `"`-inside-a-string literal, so `scanStr`/emit need no escape handling:
  1. **Bare, quote-free Core IR prims** instead of `__arith__ "sym"`/`__method__ "name"`: `+`‚Üí`i.add`,
     `-`‚Üí`i.sub`, `*`‚Üí`i.mul`, `<`‚Üí`i.lt`, `==`‚Üí`__eq__` (polymorphic int+string), string concat
     `++`‚Üí`sconcat`, `.length`‚Üí`slen`, `.charAt`‚Üí`scodeAt`, `.substring`‚Üí`sslice`. None emit a quoted operator
     string. (All verified to run.)
  2. **A `dq` parameter** carries the `"` character. `compile(src, dq)` ‚Äî the ssc0 driver builds `dq` via
     `#sfromCodes(Cons(34, Nil))` (proven: wraps `hello` in real quotes) and passes it in; C emits
     `dq ++ content ++ dq` for `(lit (str ‚Ä¶))`. So C's source uses `dq` (a value), never a `"` literal, and
     `scanStr` compares the char **code** `34`, never a `"` literal.

  Result: C's source has only quote-free string literals (the Core IR fragments `(prim i.add `, `(local `,
  `) `, ‚Ä¶) ‚Äî no `\"`, no escapes, no decode. Self-compilation never sees an internal `"`.

## Phasing

- **P6.6a ‚Äî driver.** An ssc0 driver reading a source file and calling an existing self-host compiler's
  `compile` (via `dropLast` of the projection's hard-coded `main` + a file-reading `main` AST built from
  `mk*`/prim `Pair`s). Proves "the compiler reads its source from a file and compiles it".
- **P6.6b ‚Äî F completeness.** Add var patterns in `match` + string escapes to the compiler; verify.
- **P6.6c ‚Äî write C in F.** Rewrite the compiler using only F (no `let ‚Ä¶ in`; local bindings via var-`match`;
  escaped string literals); confirm the spike still compiles it byte-identically and it still emits correct
  executable Core IR.
- **P6.6d ‚Äî fixpoint.** Bootstrap C0; run stage1/stage2; assert `stage1 == stage2`. Add to the harness.

## Relationship to P6.5

P6.5 proved every capability + built a compiler whose object language is feature-complete for the
implementation language (`selfhost-{scala,multi,env,match,str}`). P6.6 is the literal fixed point: the same
compiler, made self-applicable (source is a valid F program) and driven over its own source file.

## RESULT ‚Äî the fixpoint holds (2026-07-14)

Delivered: `specs/v2.2-p6.6-cmin.L` ‚Äî **C_min**, a compiler `compile(src, dq): String` for language L, written
in L, whose whole source is inside L (74 defs). Design choices that made a ~74-line compiler correct-by-parts:

- **match only for `Cons/Nil/(a,b)`; ALL token-kind dispatch via `if`-chains + `fst/snd/hd/tl`.** No
  int-literal patterns `(2,26)`, no `case _`, no nested non-trivial matches ‚Äî so C_min's own `parseArms` only
  has to handle the three patterns it actually uses.
- **helper-function bindings** instead of `val`-blocks (`val x=e; body` ‚Üí `def h(‚Ä¶,x)=body; h(‚Ä¶,e)`).
- **escape-free** via a `dq` PARAMETER carrying `"`; bare quote-free prims (`i.add/i.sub/i.mul/i.lt`,
  `__eq__`, `sconcat`, `slen/scodeAt/sslice`); `+`‚Üí`i.add`, `++`‚Üí`sconcat` kept distinct (`i.add` is not
  polymorphic on strings, but `__eq__`/`sconcat` auto-stringify an int operand).
- **only `<` and `==`** for numeric compare (`a>=b` ‚â° `b<a+1`).

Bootstrap + fixpoint (`specs/v2.2-p6.6-fixpoint.sh`, self-contained ‚Äî ssc1-front only, no sbt/spike):

```
C0     = driver(ssc1-front(cmin.L))   -- reference front lowers C_min; wrap with a file-reading main
stage1 = C0(cmin.L)                    -- C_min compiles its OWN source (balanced Core IR, 22085 B)
C1     = stage1 + the same file-main   -- the self-produced compiler, made runnable
stage2 = C1(cmin.L)
ASSERT   stage1 == stage2              -- byte-identical ‚úì
```

Checks in the script: 12 L conformance programs compile+run to the right value; `C1` is itself a working
compiler (compiles `fac(5)`‚Üí120); `stage1 == stage2` byte-for-byte. **No quine** ‚Äî the compiler reads its
source from a file via `#io.args()`/`#io.readFile`; nothing is embedded in a literal. The one bug found and
fixed: C_min must emit `true`/`false` as `(lit true)`/`(lit false)` ‚Äî otherwise `false` parsed as an unbound
local ‚Üí `(local 0)` (a char code), which ssc1-front had silently masked in the bootstrap.

The `dq` mechanism, concretely: the driver builds the quote as a string literal `mkStr(#sfromCodes(Cons(34,
Nil)))`; `#coreir.encode` escapes it to `\"` and `run-ir` reads it back as one `"`. So C_min emits a string
literal as `dq ++ content ++ dq` and its own source never contains a `"`-inside-a-string.

Capstone ‚Äî the **spike** bootstraps C_min (2026-07-14, DONE): `cmin` is a toy in `ScalaSpikeSpec` (reads
`specs/v2.2-p6.6-cmin.L`), and the p6.0 harness (`specs/v2.2-p6.0-spike-verify.sh`) confirms
**`spike(cmin.L) ‚â° ssc1-front(cmin.L)` byte-identical** (both 42135 B). So the UniML ScalaScript dialect
parses+projects C_min's entire 74-def source identically to the v2 reference front ‚Äî the spike is itself a
valid bootstrap compiler for C_min, tying the fixpoint to the p6.0 differential invariant. (cmin has no
runnable `main` ‚Äî it is a compiler ‚Äî so the harness reports `CoreIR‚â°ssc1-front` with an empty run-ir line;
the byte-identity is the assertion.)

## P6.7 ‚Äî L gains `val`-blocks (let); C_min self-uses them (2026-07-14, DONE)

L was extended with **braced `val`-blocks**: `{ val x = e1  val y = e2  finalExpr }` ‚Üí nested lets
`(let (e1) (let (e2) finalExpr))` (bound var at local0, outer locals shift +1 ‚Äî the `Cons(name, env)` /
`idxOf` convention). Three parts, all landed, fixpoint preserved:

- **C_min compiles them** ‚Äî new `parseBlock`/`parseBlockVal`/`parseBlockEnd` + a `{`(28) atom case + `val`(7)
  in `kwCode`. `run-ir` supports the `(let (E) BODY)` form directly.
- **C_min self-uses them** ‚Äî 5 of C_min's `‚Ä¶2`-suffixed helper functions (the ones split *only* to avoid
  `val`) were merged back into idiomatic `{ val ‚Ä¶ ; expr }` blocks (`parseArmTup`, `parseArmCons`,
  `postfixDot`, `climbStep`, `parseOneDef`). C_min is now 72 defs. **The fixpoint still holds**: `stage1 ==
  stage2` byte-identical (22794 B), C1 compiles `fac(6)`‚Üí720. So self-compilation now exercises the new
  block path.
- **The spike parses them** ‚Äî `ScalaSpike.parseAtom` gained a `spike.lbrace` case (`parseBracedBlock`)
  producing the SAME `spike.block` node as an offside block, so it projects byte-identically to ssc1-front.
  Before this, a braced block projected to `(prim __notImplemented__)` (the spike only did *offside*
  blocks). Regression: `braced-block`/`braced-nest` toys + a unit test in `ScalaSpikeSpec`; p6.0 harness 88
  ok / 0 FAIL; **`spike(cmin.L) ‚â° ssc1-front(cmin.L)` byte-identical (43898 B)** on the val-block C_min.

## P6.13 ‚Äî L gains comparison + boolean operators; C_min self-uses them (2026-07-14, DONE)

L was extended with `>`/`>=`/`<=` (‚Üí bare prims `i.gt`/`i.ge`/`i.le`) and short-circuit `&&`/`||`
(desugared in `emitBin` to `(if L R (lit false))` / `(if L (lit true) R)` ‚Äî no new prim). C_min's lexer
gained `lexLt`/`lexGt`/`lexAmp`/`lexPipe` and `binPrec` was renumbered (`||`1 `&&`2 comparison 3 `+ -`4
`*`5, preserving all relative precedences). **C_min now self-uses them** ‚Äî its character classification is
idiomatic: `isLo(c) = c >= 97 && c <= 122` (was `if 96 < c then c < 123 else false`), `atEnd(i, n) = i >= n`.
C_min is now 76 defs; the fixpoint STILL holds (`stage1 == stage2`, 25234 B; C1 compiles `fac(6)`‚Üí720), and
`spike(cmin.L) ‚â° ssc1-front(cmin.L)` remains byte-identical (46697 B) ‚Äî the spike already lowered these
operators (P6.9/ops-bool), so no spike change was needed.

## P6.14 ‚Äî L gains match wildcard `case _`; C_min self-uses it (2026-07-14, DONE)

L's `match` gained a wildcard/default arm `case _ => body`, lowered to CoreIR `(match scrut (arms)
(default body))` ‚Äî the `(default ‚Ä¶)` sits OUTSIDE the arm list. C_min's lexer tokenizes `_` (code 95 ‚Üí
`(2,45)`); `parseArms` was restructured to return `(arms, (defStr, rest))` (a nested Tuple2, since C_min has
no 3-tuple), with every arm helper threading the default+rest pair; a `case _` arm emits the `(default ‚Ä¶)`
string AND consumes the match's closing `}` (missing that skip first leaked raw token codes as garbage def
names). **C_min self-uses it** ‚Äî `hd`/`tl`/`isEmpty` now dispatch with `case _` instead of a redundant
`Cons`/`Nil` second arm. C_min is now 77 defs; the fixpoint holds (`stage1 == stage2`, 25979 B; C1 compiles
`fac(6)`‚Üí720), and `spike(cmin.L) ‚â° ssc1-front(cmin.L)` stays byte-identical (48663 B) ‚Äî the spike already
handled `case _` (the p6.0 `match-lit` toy), so no spike change was needed.

## P6.15 ‚Äî L gains int-literal match patterns `case N =>`; C_min self-uses them (2026-07-14, DONE)

L's `match` gained integer-literal arms ‚Äî `n match { case 0 => ‚Ä¶ case 1 => ‚Ä¶ case _ => ‚Ä¶ }`. C_min detects
an int-literal match in `postfixMatch` (the first arm's pattern token is kind 0, an int literal) and emits an
INLINE if-chain `(if (prim __eq__ recv (lit (int N))) body <rest>)` ending in the `case _` default ‚Äî a
distinct path (`parseIntMatch`/`parseIntArms`/`parseIntArm`) from the ctor-pattern match. (ssc1-front instead
let-binds the scrutinee once; C_min emits its own bare-prim style, and since the scrutinee is a simple local
in practice, an inline chain is correct ‚Äî only self-consistency + runnability matter for the fixpoint.)
**C_min self-uses it** ‚Äî `arithBare(k)` now dispatches token codes with `k match { case 23 => "i.add" ‚Ä¶ case _
=> "i.le" }` instead of an if-chain. C_min is now 80 defs; the fixpoint holds (`stage1 == stage2`, 27586 B; C1
compiles both `fac(6)`‚Üí720 and an int-literal program‚Üí1), and `spike(cmin.L) ‚â° ssc1-front(cmin.L)` stays
byte-identical (52138 B) ‚Äî the spike already handled int-literal patterns (p6.0 `match-lit`).
