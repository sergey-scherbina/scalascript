Status: **implementation in progress** — Steps 0-3 landed
(2026-05-19), minimal fixture green end-to-end, full `distributed-*`
suite (Steps 4-7) still in flight on `feature/dep-cps`. Spec

## 1. TL;DR

User-written `.ssc` code on the JVM backend is automatically
**CPS-rewritten** by `JvmGen` whenever it touches an effect primitive
(`Async.*`, `Actor.*`, `Random.*`, `Storage.*`, `Clock.*`, …). The
rewriter walks effectful expressions and wraps each primitive call in
`_bind(call, result => continuation)` so the synthetic `_Perform` Free
node returned by the primitive is handed to the right interpreter
(`_runActors`, `_runRandom`, …) instead of being treated as a value.

Dep code — Scala source inlined from `[name](./dep.ssc)` markdown-link
imports — **bypasses this rewriter entirely**. `inlineImport` parses
the dep, packages its blocks into `object pkg { object sub { … } }`,
and emits them verbatim. Any effect primitive call inside a dep
function therefore returns a raw `_Perform` value which the dep code
uses as if it were the resolved result. The dep ships a CPS-incorrect
program; user code that calls into the dep then crashes with
`ClassCastException: _Perform cannot be cast to <X>` (when the
`_Perform` is used as a value of type X) or silently misbehaves (when
the `_Perform` is in statement position and its effect — e.g. sending
a message, blocking on `receive` — is dropped on the floor).

The six v1.22 distributed-* conformance tests
(`distributed-{map,shuffle,failure-retry,failure-partial,heterogeneous}`,
`cluster-connect`) all hit this. Surface compilation now succeeds
(see §5 for the band-aids that got us there); the runtime layer is
the open problem.

**Recommended fix:** introduce a **dep-mode CPS rewriter** in `JvmGen`
that walks every `Defn.Def` body inside an inlined import, identifies
calls whose callee is in a known **effect-primitive set**, and rewrites
the surrounding control flow into `_bind` form — turning the dep
function's return type from `T` into a Free node that the caller's
`_bind` (already generated by the user-code rewriter) drives to
completion.

Estimated cost: 2–3 focused days. The blockers are subtle (overly broad
predicates regress unrelated tests; CPS rewriting of nested case bodies
needs care) but well-localised in `backend-jvm/src/main/scala/scalascript/codegen/JvmGen.scala`.

## 2. Concrete failure trace

Reproducer (already in the repo as `conformance/distributed-map.ssc`):

```scalascript
runActors {
  val doubleFn: Int => Int = (x: Int) => x * 2
  HandlerRegistry.register(NamedHandler("double", doubleFn))

  val w1 = spawn { () => WorkerProtocol.handleMessages() }
  val w2 = spawn { () => WorkerProtocol.handleMessages() }
  val w3 = spawn { () => WorkerProtocol.handleMessages() }

  val nodes:   List[Node] = List(Node("n1"), Node("n2"), Node("n3"))
  val cluster: Cluster    = Cluster(nodes, List(w1, w2, w3))

  val stage  = Stage(MapOp(named("double")))
  val result = runDistributed(
    data = List(1, 2, 3, 4, 5, 6),
    stage = stage,
    cluster = cluster,
    retries = 0,
    allowPartial = false
  ).asInstanceOf[DistributedResult[Any]]

  println(result.results.asInstanceOf[List[Int]].sorted)
}
```

`runDistributed` lives in `std/mapreduce/distributed.ssc` and starts with:

```scalascript
def runDistributed(
  data: List[Any], stage: Stage, cluster: Cluster,
  retries: Int, allowPartial: Boolean
): DistributedResult[Any] =
  val jobId: String = Random.uuid().asInstanceOf[String]      // ← line A
  …
  val me = self()                                              // ← line B
  trapExit(true)                                               // ← line C
  …
  val workers = cluster.pids.zip(cluster.nodes).map {
    case (wPid, node) =>
      link(wPid)                                               // ← line D
      (wPid, node)
  }
  …
  val assignments = workers.zip(partitions).zipWithIndex.map {
    case (((wPid, node), partition), idx) =>
      wPid ! ProcessPartition(jobId, idx, partition, stage, me) // ← line E
      (idx, wPid, node)
  }
  collectResults(jobId, assignments, …)
```

`collectResults` then loops on `receive { case PartitionResult(…) => …
case PartitionFailed(…) => … case Exit(…) => … }`.

After the JVM backend's preamble rewriting and dep inlining, the
emitted Scala for line A looks like:

```scala
def runDistributed(data: List[Any], stage: Stage, cluster: Cluster,
                   retries: Int, allowPartial: Boolean): DistributedResult[Any] = {
  val jobId: String = Random.uuid().asInstanceOf[String]
  …
}
```

`Random.uuid()` is defined in the preamble as:

```scala
object Random:
  def uuid(): Any = _perform("Random", "uuid")    // ← returns _Perform
```

so `Random.uuid()` is a `_Perform("Random", "uuid", Nil)` Free node.
`asInstanceOf[String]` on it raises:

```
java.lang.ClassCastException: class …$_Perform cannot be cast to class
java.lang.String
    at runDistributed(distributed-map.sc:7964)
```

That is the **first** failure. If we patched just this line (e.g.
substituted a sync `_uuidSync()` helper), control would reach line B
where `self()` would silently return another `_Perform`, then line C
(`trapExit(…)` drops a `_Perform` in statement position — `trapExit`
is **never** applied), then line D (links never installed → `Exit`
messages never delivered → worker crashes look like timeouts), then
line E (`!` returns `_Perform` instead of sending — the message is
never enqueued, the worker never receives, `collectResults` blocks
forever on `receive { … }`, which itself returns a `_Perform` whose
continuation is never invoked → permanent hang or NPE depending on
which surrounding statement first tries to use the discarded value).

In short: **`distributed-map.ssc` is structurally broken from the
first effect primitive onwards**. The CCE is just the loudest of a
cascade of identical bugs.

## 3. Why user code works but dep code doesn't

### 3.1 User-code CPS rewriting

`JvmGen.genModule` walks each user code block. For every expression
whose evaluation touches the effect-primitive surface, it calls
`emitCpsExpr` instead of the regular `emitExpr`. `emitCpsExpr`:

1. Recognises calls to known effect primitives (`Async.delay`,
   `Actor.send`, `Random.uuid`, `Storage.get`, `pid ! msg` infix,
   `receive { case … }`, `receiveWithTimeout(t) { case … }`, …).
2. For each such call inside a larger expression, generates
   `_bind(call, (x: T) => continuation_emit)` where `continuation_emit`
   is the recursive CPS emit of the rest of the expression. The result
   is a Free-monad chain ending in a single value (or `Unit`) at the
   leaves.
3. Walks sequencing constructs (`val x = e1; e2`, `if`/`else`, `match`,
   `for`, function literals, partial-function case bodies) and
   threads the continuation through correctly so the user can write
   straight-line code and the rewriter produces the CPS chain
   behind the scenes.

The output of `runActors { … }` therefore is something like:

```scala
_runActors(() =>
  _bind(_perform("Random", "uuid"), (jobId: Any) =>
    _bind(_perform("Actor", "self"), (me: Any) =>
      _bind(_perform("Actor", "trapExit", true), (_: Any) =>
        _bind(_perform("Actor", "link", w1), …)))))
```

`_runActors` drives the chain by stepping through each `_Perform`,
dispatching to the registered handler (queue a send, suspend on
receive, generate UUID, …), and resuming the continuation with the
result.

### 3.2 Dep blocks bypass the rewriter

`inlineImport(path)` (JvmGen.scala ~line 753) does:

```scala
private def inlineImport(path: String): (List[JvmGen.Block], List[String]) =
  …
  val importedModule = Parser.parse(os.read(resolved))
  val pkg = importedModule.manifest.flatMap(_.pkg).getOrElse(Nil)
  val nested = new JvmGen(Some(resolved / os.up), lockPath = lockPath)
  val rewrittenBlocks = nested.collectBlocks(importedModule.sections)
    .map(b => qualifyBareActorCallsInBlock(b))      // ← textual rewrite only
  (rewrittenBlocks, pkg)
```

`collectBlocks` returns the dep's code blocks as opaque `Block(node,
src)` values. They flow through the regular `emitBlock` path. But
`emitBlock` only calls `emitCpsExpr` when an expression's outermost
form is recognised as effectful — and a `Defn.Def(name, …, body)` is
not an effectful expression: it's a *definition* whose body is type
`T`, opaque from `emitBlock`'s perspective. So the body emits via
the regular non-CPS `emitStat` / `emitExpr` path, which faithfully
reproduces the Scala source verbatim.

Result: a dep function's body is emitted as plain Scala. When the
body says `Random.uuid()`, the emit just prints `Random.uuid()`. The
runtime call returns `_Perform`, which the body uses as a value.

### 3.3 Why this isn't a parser bug

The dep author wrote `val jobId: String = Random.uuid()`. The
expression *should* CPS-expand to:

```scala
_bind(Random.uuid(), (jobId: String) => <rest of function>)
```

and the function itself should become `def runDistributed(…): Any =
<that bind chain>`, returning a `_Perform` node that the caller's
`_bind` will drive. That's the rewrite that fires automatically on
user code but skips dep code.

The dep author can't write the `_bind` chain by hand — that's exactly
the boilerplate the rewriter exists to eliminate. So the dep author
writes straight-line effect-primitive calls and the backend silently
emits a broken program.

## 4. Design space

Four roughly-distinct strategies, ordered from least to most invasive.

### 4.1 Strategy A — sync helpers for pure primitives

**Idea:** for primitives that have no real "effect" beyond observation
(UUID, current time, RNG draws), add `_uuidSync()`, `_nowSync()`,
`_nextIntSync(n)` runtime helpers and rewrite dep calls to use them.

**Scope:** ~50 LOC of runtime additions plus a rewriter pass.

**Why it doesn't fix the problem:** `Random.uuid()` is the *first*
broken call in `runDistributed`, but `self()`, `link()`, `pid ! msg`,
`receive { … }`, `trapExit(…)` cannot be made synchronous. They need
the actor scheduler. So Strategy A unblocks exactly the CCE on line A
and leaves lines B–E and the receive loop broken. Not viable as a
standalone solution; could be a future micro-optimisation once the
main problem is solved.

### 4.2 Strategy B — split dep into "interface" and "implementation"

**Idea:** keep type-only declarations in the dep `.ssc`; force the
implementation into user code so it goes through the user-code
rewriter.

**Scope:** API-level — every `def runDistributed` would have to be
copy-pasted into each test (or imported via a non-`.ssc` mechanism
that doesn't trigger dep inlining).

**Why it doesn't fix the problem:** defeats the purpose of having
deps. The whole point of `std/mapreduce/distributed.ssc` is that
user code says `runDistributed(…)` once. Inlining the body into
every test undoes modularity. Reject.

### 4.3 Strategy C — runtime preamble for `std/mapreduce`

**Idea:** treat `std/mapreduce/*` as runtime, not as a dep. Hand-write
the CPS-correct implementation of `runDistributed`, `collectResults`,
`handleFailure`, `WorkerProtocol.handleMessages`, etc. directly in
the JvmGen runtime preamble — same way `Actor`, `Random`, `_runActors`
are emitted today.

**Scope:** ~300–500 LOC of hand-written Scala in `JvmGen.scala`. Plus
a gate: only emit the `JvmRuntimeMapReduce` preamble when the IR
references one of its types (analogous to `JvmRuntimeDataset`).

**Pros:** the implementation is written once, by hand, in CPS form,
and is known-correct. No rewriter changes needed. The dep `.ssc`
files become specification documents that describe the API; the
real implementation lives in the backend.

**Cons:** every backend (JS, INT, future WASM) has to re-implement
the runtime separately. The dep `.ssc` becomes lying documentation
unless we add a test that the runtime preamble matches the dep file
behaviourally. Significant duplication across backends. Loses the
v2.0 "one source, many backends" property for `std/mapreduce`
specifically.

**Verdict:** viable fallback if Strategy D proves too risky. The
JS backend will need this anyway (its CPS story is different), so
the duplication argument is partially moot.

### 4.4 Strategy D — dep-mode CPS rewriter (RECOMMENDED)

**Idea:** extend the user-code CPS rewriter so it also fires on dep
function bodies, but with a *tight predicate* — only wrap calls whose
callee is in the known effect-primitive set, leave the rest of the
expression alone.

**Scope:** ~150–300 LOC in `JvmGen.scala`, plus careful test
coverage. Three sub-pieces:

1. **Dep-mode emit path.** When `emitBlock` runs inside an
   inlined import, mark the emit context as `depMode = true`.
   In dep mode, `Defn.Def(name, params, retType, body)` becomes
   `Defn.Def(name, params, Any, emitCpsExpr(body))` *iff*
   `containsEffectPrimitive(body)` returns true. Otherwise emit
   unchanged (so the cost is paid only by effectful defs).

2. **`containsEffectPrimitive(tree)` predicate.** Walks the tree
   structurally looking for any leaf in a tight whitelist:
   - `Term.Apply(Term.Select(Term.Name("Random"|"Actor"|"Async"|
     "Storage"|"Clock"|…), Term.Name(op)), _)` where `op` is in
     the per-effect set.
   - `Term.Apply(Term.Name(n), _)` where `n` is in `actorBareNames`
     (`self`, `link`, `register`, `connectNode`, …) — these are
     bare aliases the user-code path rewrites; dep mode handles
     them the same way after `qualifyBareActorCallsInSource`.
   - `Term.ApplyInfix.After_4_6_0(_, Term.Name("!"), _, _)` — actor
     send sugar.
   - `Term.Apply(Term.Name("receive" | "receiveWithTimeout"), …)`
     with a `PartialFunction` argument — receive blocks.
   - Recursive: a call to any other dep-defined `def` (in *any*
     transitively inlined dep, bare or qualified) whose body itself
     contained a primitive. Computed once per `ssc compile` via a
     fixpoint pass over the union of all dep call graphs, before
     any emit happens. See §8.2 for why per-compile rather than
     persisted-in-`.scim`.

3. **CPS emit reuse.** `emitCpsExpr` is the same as today — same
   `_bind` plumbing, same `emitReceiveMatcherOpt`. Dep mode just
   widens the function's declared return type to `Any` (the
   `_Perform` is a subtype of the runtime's `_Computation` /
   `Any`) so the caller's `_bind` accepts it. The caller is
   user-code, already CPS-rewritten, already wraps the result of
   `runDistributed(…)` in `_bind(_, result => …)`, so the chain
   composes.

**The hard part — predicate tightness:**

The earlier attempt at structural recursion regressed
`actors-process-info.ssc`. Specifically, `info.links.length` inside
`case Some(info) => info.links.length` was getting wrapped as
`_bind(info.links, x => x.length)`. The bug: `termNeedsCustomEmit`
matched any `Select` on an `Any`-typed receiver (the case binding
`info: Any` because `Some.value: Any` after erasure), and the
broad effect-detection fired.

**Fix:** the dep-mode predicate must be call-site shape based, not
type-inference based. A `Select` is *never* effectful by itself. A
`Term.Apply(Term.Select(Term.Name("Random"), Term.Name("uuid")),
Nil)` *is*. The whitelist is closed and small (~30 shapes total
across Actor / Async / Random / Storage / Clock / receive). Match
those exactly; everything else is non-effectful even if its type
infers to `Any`.

**Sequencing rule:** if a `Block(stats, expr)` contains *any*
effectful sub-expression, the whole block CPS-emits. The cheap
non-effectful stats become `_bind(_pure(_), _ => …)` or just inline
under the continuation — either is correct. Today's `emitCpsExpr`
already handles this; dep mode reuses it.

**Function-call propagation:** if `def f` is effectful and `def g`
calls `f`, then `g` is effectful. The fixpoint pass propagates
effectfulness through the dep's internal call graph. External
(non-dep) calls assume non-effectful unless the callee is in the
primitive whitelist.

**Verdict:** the right fix. Preserves modularity, doesn't duplicate
runtime across backends, scales to future deps that compose effect
primitives. The implementation cost is honest 2–3 days because the
fixpoint pass + predicate tuning + regression-proofing each need
careful test coverage. But there are no fundamental unknowns left.

## 5. What's already landed (band-aids, all retained)

Three layers of textual / AST rewriting got `distributed-map.ssc`
from 27 compile errors → 0 compile errors. They don't fix runtime
but they're prerequisites for Strategy D to even be testable, so
they stay.

### 5.1 `qualifyBareActorCallsInSource(src: String): String`

Regex-based rewriter applied to every dep block before it enters the
emit pipeline. For each name in `actorBareNames - {receive, runActors,
spawn, spawn_link, spawnBounded}`, replaces `<name>(` with
`Actor.<name>(` when the preceding character isn't `.` / word-char
(so `Actor.self(` and arbitrary identifiers aren't double-qualified).

Why string-level, not scalameta `Tree.transform`: scalameta 4.13
ships no public `Tree.transform` (the `transversers` package is
empty), and writing a manual recursive copy-based transformer for
every container type would be ~200 lines of plumbing for a rewrite
this surgical. The regex set is intentionally tiny.

### 5.2 `rewriteActorAstCallsInSource(src: String): String`

scalameta-based AST walk for shapes the regex can't safely express:

- `pid ! msg` infix → `Actor.send(pid, msg)`. Reconstructed
  syntactically for `pid ! (a, b)` because Scala parses that as a
  2-arg infix application, not as a 1-arg send of a tuple.
- `receive { case … }` and `receive(t) { case … }` /
  `receiveWithTimeout(t) { case … }` → `Actor.receive_(…)` /
  `Actor.receive_t(…, t)` with a registered matcher spec. Reuses
  `emitReceiveMatcherOpt(cases, cpsBody = false)`; the `cpsBody =
  false` path recursively reapplies the whole rewriter to each case
  body so the outer receive's splice doesn't clobber inner `!`
  splices.

Splices computed right-to-left from `pos.start/end` so earlier
offsets stay valid as later replacements shift the source.
Source-parse tried first, then `Term`-parse fallback so block-shaped
case bodies also walk.

### 5.3 `emitCpsBindWithType(rhs, name, tpe, body)`

When the user writes `val result: DistributedResult[Any] =
runDistributed(…)`, the CPS rewriter emits:

```scala
_bind(runDistributed(…),
  ((result: DistributedResult[Any]) => …continuation…)
    .asInstanceOf[Any => Any])
```

The `.asInstanceOf[Any => Any]` widening matters: `_bind` is typed
`(Any, Any => Any) => Any`, so a `(DistributedResult[Any] => …)`
lambda would be rejected by the Scala type checker. The cast pushes
the type check to runtime, where the actual binding is dynamic
anyway.

Without this widening the test wouldn't even *compile* (let alone
crash at runtime). It's a precondition for Strategy D to land.

### 5.4 `mergeDuplicatePackageObjects` (recursive)

When several deps share a `package: std.mapreduce` prefix, each one
emits its own `object std { object mapreduce { … } }` wrapper. Scala
3 refuses to compile duplicate top-level objects. The merger is a
brace-balanced string scanner (string-literal and `//`-comment aware)
that collapses same-name siblings into one. The recursive variant
re-runs after each merge so nested duplicates also flatten.

Why not scalameta: parsing fails on the 300 KB+ emitted preamble;
the string scanner is robust enough for the controlled output shape.

## 6. Recommended implementation plan

Sequenced for incremental landing. Each step compiles and runs
existing tests green; later steps build on earlier ones.

### Step 0 — minimal fixture (`tests/fixtures/dep-cps-basic.ssc`)

Before touching JvmGen, build a deliberately tiny fixture that
exercises the four shapes Strategy D *must* get right and nothing
else:

1. Direct primitive call (`!`, `receive`).
2. Transitive effectfulness (`pingTwice` → `ping`).
3. Val binding of an effectful result.
4. Pure tail (`first + "/" + second`) embedded in a CPS chain.

Sketch (see §6.0a below for the full file):

```scalascript
def ping(pid: Any): String =
  pid ! "ping"
  receive { case "pong" => "ok" }

def pingTwice(pid: Any): String =
  val first  = ping(pid)
  val second = ping(pid)
  first + "/" + second
```

```scalascript
runActors {
  val responder = spawn { () =>
    receive { case ("ping", from) => from ! "pong" }
  }
  println(pingTwice(responder))
}
```

This fixture is the **canonical green target for Steps 1–4**. Only
after it passes do we point Strategy D at `distributed-map.ssc`
(which adds link/trapExit/recursion/3-arm-receive/mutual-recursion
on top of the same fundament). Debugging a 15-line fixture is
tractable; debugging 300 lines of `runDistributed` is not.

Acceptance: file exists, currently fails with the same
`_Perform cannot be cast to String` shape as `distributed-map.ssc`.
(If it doesn't fail, the fixture isn't actually exercising the
bug — adjust until it does.)

### Step 1 — `containsEffectPrimitive(tree)` predicate

File: `backend-jvm/src/main/scala/scalascript/codegen/JvmGen.scala`.

Add a pure predicate function that walks a `scala.meta.Tree` and
returns true iff it contains any call-site shape from the
**effect-primitive whitelist** (§4.4 sub-piece 2). Unit test the
predicate exhaustively before wiring it anywhere — its tightness is
the load-bearing property.

The predicate must match **both** bare and qualified call shapes
so cross-dep calls (Step 2) feed it correctly:

- `Term.Apply(Term.Name(n), …)` where `n` is a bare alias
  (`actorBareNames`).
- `Term.Apply(Term.Select(Term.Name(modOrObj), Term.Name(op)), …)`
  where `(modOrObj, op)` is in the primitive whitelist
  (`Actor.self`, `Random.uuid`, etc.) **or** where
  `modOrObj` is another dep's package object name and `op` is in
  the cross-dep `effectfulDefs` set (resolved at Step 2 time).
- `Term.ApplyInfix.After_4_6_0(_, Term.Name("!"), _, _)` — actor send.
- `Term.Apply(Term.Name("receive" | "receiveWithTimeout"), …)` with
  a `PartialFunction` argument.

Forbidden matches (regression canary, see §4.4 "The hard part"):

- Bare `Term.Select(_, _)` — never effectful on its own.
- `Term.Apply` of a user-defined function whose name is **not** in
  any known set.
- Anything matched by inferred type (we deliberately ignore types).

Acceptance: ~50 unit tests covering positive (each primitive shape
matches) and negative (`Select` on `Any`, `Apply` of user-defined
function, lambda body, pattern guard, etc. — *don't* match) cases.

### Step 2 — two-phase `inlineImport` + global fixpoint

Restructure dep ingestion into **two phases**:

1. **`collectDep(path)`** — parses, indexes all top-level `Defn.Def`
   nodes by their fully-qualified name (`pkg.subpkg.defName`), and
   records each def's body for later effectfulness analysis. **Does
   not emit anything.** Recurses into transitively imported deps
   first so the full graph is available before any emit decision.

2. After all transitive `collectDep` calls finish: build a single
   call graph over **all** ingested defs. Run fixpoint:
   - Seed: every def whose body contains a primitive (per
     `containsEffectPrimitive`) → mark as effectful.
   - Iterate: any def that calls (bare or qualified) an
     already-marked def → mark as effectful.
   - Terminate when a full pass produces no new marks.
   - Result: `globalEffectfulDefs: Set[String]` (FQN-keyed).

3. **`emitDep(path)`** — emits using `globalEffectfulDefs` as
   read-only context.

The current `inlineImport` is a single-phase DFS; this is a
mechanical refactor — `importedFiles: Set[String]` already tracks
visited paths; we just split the "what to do per-file" into
collect-then-emit.

Cost is O(deps × defs × calls), <1 ms for a corpus of ~5 deps
× ~20 defs each. No persistence needed.

Acceptance: 3-function chain fixture (`f calls g (qualified
B.g) calls Actor.self()`) — all three marked. Sibling
`pureHelper(x) = x + 1` — not marked. Cross-dep call (dep A's
`f` calls dep B's `g`) — both marked.

### Step 3 — dep-mode `Defn.Def` emit

In the dep emit path, when emitting `Defn.Def(name, …, body)` and
`globalEffectfulDefs.contains(fqn)`, route `body` through
`emitCpsExpr` instead of `emitExpr`. **Widen the declared return
type to `Any`** (resolved per §8.1: matches existing convention,
handles mixed pure/effectful control-flow correctly).

Acceptance: the fixture from Step 0 runs green end-to-end on JVM.

### Step 4 — caller-side type widening

User code calls `val r: DistributedResult[Any] = runDistributed(…)`.
With Step 3, `runDistributed` now returns `Any`. The user-code CPS
emitter already handles this via `emitCpsBindWithType` (§5.3).
Verify nothing in §5.3 needs updating; in particular the
`.asInstanceOf[Any => Any]` widening is what makes the chain
type-check.

Acceptance: `distributed-map.ssc` compiles clean (unchanged) AND
runs to a passing assertion.

### Step 5 — regression sweep

Run the full conformance + ScalaTest suite. The known regression
risk is `actors-process-info.ssc` (the historical canary for the
overly broad predicate). Add it as a required-green test for this
work.

Acceptance: zero regressions across `sbt test` + full conformance.

### Step 6 — turn on the remaining v1.22 tests

Strip the `pending:` markers from `distributed-shuffle.ssc`,
`distributed-failure-retry.ssc`, `distributed-failure-partial.ssc`,
`distributed-heterogeneous.ssc`, `cluster-connect.ssc`. Run. Fix
whatever Strategy-D corner cases each one exposes (likely: deeply
nested receive bodies, `for`-comprehensions over effectful steps,
mixed sync/async sequencing in `match` arms).

Acceptance: all six v1.22 distributed-* tests green on JVM.

### Step 7 — clean up

Two cleanups, both gated on Step 6 staying green:

1. **Remove `cpsBody` parameter from `emitReceiveMatcherOpt`**
   (resolved per §8.3). All call sites switch to the uniform
   `cpsBody = true` path through `emitCpsExpr`. The `cpsBody =
   false` branch and its recursive `qualifyBareActorCallsInSource`
   re-application become dead code.

2. **Retire §5.1's `qualifyBareActorCallsInSource` and §5.2's
   `rewriteActorAstCallsInSource`** *only if* the dep-mode CPS
   emitter subsumes their work. They may still earn their keep as
   pre-normalisation passes (turning `pid ! msg` into a regular
   `Actor.send` call lets the CPS emitter use the existing
   `Actor.send` whitelist entry rather than special-casing infix).
   Keep them if removal causes any test failure; document the
   load-bearing ones explicitly.

## 7. Cross-backend implications

JS and INT backends share the architectural problem in principle —
their dep inlining also bypasses any effect rewriting. In practice:

- **JS** doesn't currently exercise distributed actors heavily; the
  v1.22 tests are marked `backends: [jvm]`. When JS gets distributed
  support, it'll need the equivalent of Strategy D for its own
  coroutine plumbing. Out of scope for this spec.

- **INT** (tree-walking interpreter) doesn't emit Scala — it
  evaluates the IR directly. Its actor primitives return real values
  by blocking the evaluator thread, so the `_Perform` mismatch
  doesn't occur. INT-marked dep code should work today; if it
  doesn't, that's a separate bug.

The JVM-specific nature of this spec reflects the JVM's CPS-as-Scala-
source compilation strategy, not a fundamental cross-backend
architectural decision.

## 8. Resolved decisions

The four originally-open questions are settled. Rationale + chosen
answer for each below. Each subsection is normative — Steps 0–7
above already assume these answers; this section is the record of
*why*.

### 8.1 Return type of effectful dep defs → `Any`

**Decision: `Any`.**

Eliminated alternatives:

- **`_Computation`** (sealed trait, currently used for `_Perform`,
  `_FlatMap`). Fails because a dep function with mixed pure/effectful
  control flow returns either kind on different dynamic paths. Trivial
  example:

  ```scalascript
  def maybeAsk(pid: Any, useCache: Boolean): String =
    if useCache then "cached"                      // ← plain String
    else { pid ! Ask; receive { case s => s } }    // ← _Computation
  ```

  `"cached"` is not a `_Computation`, so declaring the function
  `_Computation` won't compile.

- **`Free[F, T]`**. Needs type-system support for effect rows
  (`Async | Random | …`); ScalaScript doesn't have that today, see
  `docs/coroutines.md` §v1.12. Out of scope.

Why `Any` works cleanly:

- `_bind: (Any, Any => Any) => Any` already pattern-matches in
  runtime: `case _: _Computation => _FlatMap(c, f)` / `case v => f(v)`.
  Both branches handled by the same call site.
- Existing convention: every preamble-defined primitive
  (`Actor.self(): Any`, `Random.uuid(): Any`, …) already returns
  `Any`. Strategy D extends, not breaks, this convention.
- Type information for tooling is *not* lost — `ExportedSymbol.tpe`
  in the `.scim` artifact still records the source-level annotation
  (`String`, `DistributedResult[Any]`, …). The widening lives only
  in the emitted JVM target; `.ssc` source and `.scim` interface
  keep the honest type.

### 8.2 Cross-dep effectfulness → per-compile fixpoint now, `.scim` persistence later

**Decision: per-compile fixpoint over the union of all transitively
inlined deps. `.scim` persistence deferred to v2.x.**

Why:

- The full call graph for a `ssc compile` invocation is small
  (single-digit deps × tens of defs × small call counts per body).
  Fixpoint runs in <1 ms — not worth caching.
- Separate-compilation (v2.0 artifacts) isn't yet wired into the
  conformance runner. `ssc compile` remains the primary path; it
  always has access to all dep sources at once.
- `.scim` is the natural place to persist `isEffectful` for the
  separate-compilation story (`ExportedSymbol` already exists at
  `ir/src/main/scala/scalascript/ir/Ir.scala:219` with `tpe`,
  `nested`, `definitionLine` — adding an `isEffectful: Boolean =
  false` field is one-line, backward-compatible). But that's a
  v2.x add-on once `check-with-iface`/`link` actually exercise it.

Implementation detail: the predicate (Step 1) must match both
`Term.Apply(Term.Name("ping"), …)` and
`Term.Apply(Term.Select(Term.Name("OtherDep"), Term.Name("ping")), …)`
shapes — easy to forget, called out explicitly in Step 1.

### 8.3 Receive matcher → `cpsBody = true` uniform

**Decision: drop the `cpsBody` parameter from `emitReceiveMatcherOpt`.
All paths use `emitCpsExpr` for case bodies.**

Why:

- The `cpsBody = false` path exists *only* for the textual band-aid
  (`rewriteActorAstCallsInSource`, JvmGen.scala:870, 882). Once
  Strategy D lands and the dep CPS-emits, that band-aid path goes
  away.
- `emitCpsExpr` no-ops correctly on pure case bodies: `case Tick =>
  count + 1` emits as just `count + 1`, because `_bind(value, f) ==
  f(value)` via the `case v => f(v)` branch in `_bind`. Zero
  overhead for non-effectful cases.
- Threading `effectfulDefs` into the matcher (the alternative) would
  give each case body individual emit choice — but the only thing
  that decision controls is "CPS-rewrite or not", and CPS-rewrite is
  always correct. So the decision is degenerate.

Step 7 includes the parameter removal + dead-code cleanup.

### 8.4 Canonical test target → tiny fixture first, then `distributed-map.ssc`

**Decision: build `tests/fixtures/dep-cps-basic.ssc` (Step 0) as
the canonical green target for Steps 1–4. Point at
`distributed-map.ssc` only after the fixture passes (Step 4).**

Why:

- `distributed-map.ssc` simultaneously exercises: `Random.uuid`,
  `self`, `trapExit`, `link`, `!`, 3-arm `receive` with nested
  sends, recursive `collectResults`, mutual recursion
  `collectResults ↔ handleFailure`, closures over `jobId` in
  pattern guards. A single failure could be any of seven
  Strategy-D features.
- The fixture (~15 lines) exercises exactly four shapes — primitive
  call, transitive effectfulness, val-bound effectful result, pure
  tail in a CPS chain — each one mapped to a Step 1–4 requirement.
- The fixture is **not a replacement** for `distributed-map.ssc`;
  it's the unit test. `distributed-map.ssc` remains the integration
  test (Step 4 acceptance), and the other four `distributed-*`
  tests are the full coverage sweep (Step 6).

## 9. What this spec does *not* cover

- The v1.22 design itself (`docs/mapreduce.md`, `docs/actors-dist.md`).
- The `inlineImport` packaging issues that §5.4's merger addresses.
  Those are textual / namespace problems; this spec is about
  effect semantics.
- The user-code CPS emitter's internals — assumed correct and
  reusable as a black box.
- JS / INT / future backend equivalents.
- Performance: the `_bind` chain is the same complexity in dep mode
  as in user-code mode, which is already the production runtime
  cost shape.
