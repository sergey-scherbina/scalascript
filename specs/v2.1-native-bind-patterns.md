# ScalaScript 2.1 native constructor bind patterns

## Overview

ScalaScript supports Scala-style bind patterns such as
`case ok @ ParseOk(value, rest, pos) => ...`. The alias names the complete
matched value while the nested constructor pattern validates its shape and
binds fields. The compatibility frontend already preserves this as `Pat.Bind`,
but the self-hosted frontend stops after the bare name and parses `@` as an
unknown expression operator, leaving the structural `_err` sentinel.

The native frontend must parse and lower constructor bind patterns directly to
portable CoreIR, without source rewriting, partial-IR execution, Scalameta, or
runtime special cases.

## Interface

```scalascript
result match
  case ok @ ParseOk(value, rest, pos) =>
    println(ok)
    println(value)
  case ParseErr(error) => println(error)
```

On success, `ok` is the entire `ParseOk/3` value and `value`, `rest`, and `pos`
are its ordinary field bindings. The source scrutinee evaluates once.

## Behavior

- [ ] `name @ Constructor(fields...)` is one parsed pattern; `@` never reaches
      expression parsing or emits `_err`.
- [ ] The alias binds the complete once-evaluated scrutinee and constructor
      fields bind in their established source order.
- [ ] Wildcard fields do not create visible names, while named fields and the
      alias are simultaneously in scope in the arm body.
- [ ] Constructor mismatch falls through to the next source arm without
      exposing the alias or field binders.
- [ ] Nested constructor obligations inside the bound outer constructor retain
      the existing ordered-fallback behavior.
- [ ] A focused fixture proves whole-value reuse, field access, wildcard fields,
      once-only scrutinee evaluation, and fallthrough identically on native VM
      and direct ASM.
- [ ] The complete imported recovery evaluator is sentinel-clear, and installed
      `examples/dsl-sql-recovery.ssc` matches explicit compatibility output on
      VM/direct ASM.
- [ ] Native-entry, full corpus/parity, both taxonomies,
      standard/slim/JRE/dependency/build-jvm, and fresh affected conformance
      gates pass before its reviewed blocker is retired.
- [ ] Scalameta, v1 execution, parser-specific runtime branches, source rewrite,
      and transparent compatibility fallback are absent.

## Out of scope

- Typed aliases, literal aliases, alternative-pattern aliases, and guards on a
  bind pattern; these require their own explicit semantics and consumers.
- Changing constructor tags, tuple representation, or nested-pattern fallback.
- Weakening the structural `_err` sentinel gate or treating it as executable.

## Design

The self-hosted pattern parser introduces a `bpat(alias, inner)` node only when
a lowercase binder is immediately followed by `@` and a constructor/tuple
pattern. The ordinary constructor parser continues to own field and nested
pattern structure.

Match lowering already evaluates the scrutinee once and routes constructor
patterns through ordered fallback. A bound constructor arm reuses that saved
scrutinee: after the outer tag matches, it introduces one `IrLet` for the alias,
keeps the existing field binders underneath it, shifts any nested-pattern
obligations by that one local, and lowers the body. The outer tag or nested
obligation failing evaluates the remaining source arms with dummy padding only;
the alias cannot leak into fallback scope.

## Decisions

- **Frontend pattern node** — chosen because aliasing is source grammar and the
  entire value is already available in the once-evaluated match local.
  Rejected: reconstructing the value from fields, which can lose producer
  identity and evaluates the wrong contract.
- **Compose with ordered constructor lowering** — chosen to retain current
  nested fallback and de Bruijn scope rules. Rejected: a VM/ASM bind-pattern
  primitive or parser-value special case.
- **Keep strict sentinel rejection** — `_err` indicates an incomplete frontend
  parse and must never become a compatibility route.

## Baseline

After multiline imports are restored, the assembled structural compiler finds
three `_err` sites, one for each `case ok @ ParseOk(_, _, _)` in
`std/parsing/recovery.ssc`. Both installed native lanes reject the document
before execution. The focused multiline-import fixture is already exact (`82`),
proving this is a separate pattern-language boundary.
