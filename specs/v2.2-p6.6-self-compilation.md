# v2.2 P6.6 ÔÇö literal self-compilation fixpoint (no quine)

Status: **in progress.** Goal: a compiler that compiles **its own source**, reaching a stage1 == stage2
fixed point. This closes the literal end of the P6.5 self-host arc.

## The correction: there is no quine

An earlier note claimed a "quine problem" ÔÇö that to feed a compiler its own source you must embed the
source as a string literal in itself. **That is wrong.** The compiler reads its source **from a file**,
exactly the way the trusted reference runner `v2/bin/ssc1-run.ssc0` reads a `.ssc` file:

```
def main = () =>
  match #io.args() {
    case Cons(path, rest) =>
      let src = #utf8->str(#io.readFile(path)) in
      #io.print(compile(src))
    case Nil => #io.eprint("usage: <file>\n")
  }
```

The runtime primitives `#io.args`, `#io.readFile`, `#utf8->str`, `#io.print` all exist (`Runtime.scala`;
`ssc1-run.ssc0` uses them). A minimal ssc0 driver reading a file and printing its content has been verified
end to end.

## The one real subtlety (the driver, not a quine)

`#`-primitives are **ssc0**, not the Scala **subset** the self-host compiler is written in
(`def main(): Int = #i.add(3, 4)` compiled through `ssc1-front` yields `(global _err)` ÔÇö the subset front
does not know `#`-prims). So the **file reading is done by an ssc0 driver** that wraps the compiler's pure
`compile: String Ôćĺ String` function; the compiler stays a pure function, the driver does the I/O. This is
exactly how `ssc1-run.ssc0` is structured (a driver around `lowerProg`).

## The fixed point

Let **C** be a compiler for a language **F**, written **in F**, emitting **executable Core IR**. Let
`driver(defs)` wrap a set of compiled `defs` (which include `compile`) in the file-reading `main` above.

1. **Bootstrap.** The spike (front + `ssc1-lower`) compiles `C`'s source (F ÔŐć subset) Ôćĺ `C0 = driver(spike(C))`.
2. **Stage 1.** Run `C0` with `C`'s own source file Ôćĺ `stage1 = C0.compile(C_src)` ÔÇö C's definitions, compiled
   by the spike-built C.
3. **Stage 2.** Run `C1 = driver(stage1)` with `C`'s own source file Ôćĺ `stage2 = C1.compile(C_src)`.
4. **Fixed point.** `stage1 == stage2` (byte-identical). This holds iff `C0.compile` correctly compiles F
   (including C's own source): a correct `C0.compile(C_src)` is a `compile` function equivalent to
   `C0.compile`, so `C1.compile(C_src) == C0.compile(C_src) == stage1`. `scalac`/the spike stays the outer
   oracle for the bootstrap.

Nothing here embeds C in itself. C's source is a **file** the driver reads at runtime.

## The language F

F must be rich enough to write C in, and F ÔŐć subset (so the spike bootstraps C). From the P6.5 work, F =
**multi-param functions, recursion, `if`/`then`/`else`, pattern `match` (`Nil`/`Cons`/tuple/**var**),
`Cons`/`Nil`/tuple construction, strings + `charAt`/`substring`/`length`/`==`/`+`(concat), integers,
polymorphic arithmetic, multi-arg calls**, and **local bindings**.

Two design choices make C's *own source* valid F **without new escape/var-pattern machinery** (all
mechanisms below are probed + proven runnable):

- **Local bindings via helper functions, not `let ÔÇŽ in`.** `let ÔÇŽ in` is not Scala subset syntax (bootstrap
  can't parse it). Instead, lift each `val x = e; body` into a helper `def h(ÔÇŽ, x) = body` called as
  `h(ÔÇŽ, e)` ÔÇö the bound value becomes a **parameter**. Uses only multi-param functions (already in F); no
  var-patterns, no offside blocks.
- **No string escapes ÔÇö a quote-free emission design.** Two tricks together mean C's source contains **no**
  `"`-inside-a-string literal, so `scanStr`/emit need no escape handling:
  1. **Bare, quote-free Core IR prims** instead of `__arith__ "sym"`/`__method__ "name"`: `+`Ôćĺ`i.add`,
     `-`Ôćĺ`i.sub`, `*`Ôćĺ`i.mul`, `<`Ôćĺ`i.lt`, `==`Ôćĺ`__eq__` (polymorphic int+string), string concat
     `++`Ôćĺ`sconcat`, `.length`Ôćĺ`slen`, `.charAt`Ôćĺ`scodeAt`, `.substring`Ôćĺ`sslice`. None emit a quoted operator
     string. (All verified to run.)
  2. **A `dq` parameter** carries the `"` character. `compile(src, dq)` ÔÇö the ssc0 driver builds `dq` via
     `#sfromCodes(Cons(34, Nil))` (proven: wraps `hello` in real quotes) and passes it in; C emits
     `dq ++ content ++ dq` for `(lit (str ÔÇŽ))`. So C's source uses `dq` (a value), never a `"` literal, and
     `scanStr` compares the char **code** `34`, never a `"` literal.

  Result: C's source has only quote-free string literals (the Core IR fragments `(prim i.add `, `(local `,
  `) `, ÔÇŽ) ÔÇö no `\"`, no escapes, no decode. Self-compilation never sees an internal `"`.

## Phasing

- **P6.6a ÔÇö driver.** An ssc0 driver reading a source file and calling an existing self-host compiler's
  `compile` (via `dropLast` of the projection's hard-coded `main` + a file-reading `main` AST built from
  `mk*`/prim `Pair`s). Proves "the compiler reads its source from a file and compiles it".
- **P6.6b ÔÇö F completeness.** Add var patterns in `match` + string escapes to the compiler; verify.
- **P6.6c ÔÇö write C in F.** Rewrite the compiler using only F (no `let ÔÇŽ in`; local bindings via var-`match`;
  escaped string literals); confirm the spike still compiles it byte-identically and it still emits correct
  executable Core IR.
- **P6.6d ÔÇö fixpoint.** Bootstrap C0; run stage1/stage2; assert `stage1 == stage2`. Add to the harness.

## Relationship to P6.5

P6.5 proved every capability + built a compiler whose object language is feature-complete for the
implementation language (`selfhost-{scala,multi,env,match,str}`). P6.6 is the literal fixed point: the same
compiler, made self-applicable (source is a valid F program) and driven over its own source file.

## RESULT ÔÇö the fixpoint holds (2026-07-14)

Delivered: `specs/v2.2-p6.6-cmin.L` ÔÇö **C_min**, a compiler `compile(src, dq): String` for language L, written
in L, whose whole source is inside L (74 defs). Design choices that made a ~74-line compiler correct-by-parts:

- **match only for `Cons/Nil/(a,b)`; ALL token-kind dispatch via `if`-chains + `fst/snd/hd/tl`.** No
  int-literal patterns `(2,26)`, no `case _`, no nested non-trivial matches ÔÇö so C_min's own `parseArms` only
  has to handle the three patterns it actually uses.
- **helper-function bindings** instead of `val`-blocks (`val x=e; body` Ôćĺ `def h(ÔÇŽ,x)=body; h(ÔÇŽ,e)`).
- **escape-free** via a `dq` PARAMETER carrying `"`; bare quote-free prims (`i.add/i.sub/i.mul/i.lt`,
  `__eq__`, `sconcat`, `slen/scodeAt/sslice`); `+`Ôćĺ`i.add`, `++`Ôćĺ`sconcat` kept distinct (`i.add` is not
  polymorphic on strings, but `__eq__`/`sconcat` auto-stringify an int operand).
- **only `<` and `==`** for numeric compare (`a>=b` Ôëí `b<a+1`).

Bootstrap + fixpoint (`specs/v2.2-p6.6-fixpoint.sh`, self-contained ÔÇö ssc1-front only, no sbt/spike):

```
C0     = driver(ssc1-front(cmin.L))   -- reference front lowers C_min; wrap with a file-reading main
stage1 = C0(cmin.L)                    -- C_min compiles its OWN source (balanced Core IR, 22085 B)
C1     = stage1 + the same file-main   -- the self-produced compiler, made runnable
stage2 = C1(cmin.L)
ASSERT   stage1 == stage2              -- byte-identical Ôťô
```

Checks in the script: 12 L conformance programs compile+run to the right value; `C1` is itself a working
compiler (compiles `fac(5)`Ôćĺ120); `stage1 == stage2` byte-for-byte. **No quine** ÔÇö the compiler reads its
source from a file via `#io.args()`/`#io.readFile`; nothing is embedded in a literal. The one bug found and
fixed: C_min must emit `true`/`false` as `(lit true)`/`(lit false)` ÔÇö otherwise `false` parsed as an unbound
local Ôćĺ `(local 0)` (a char code), which ssc1-front had silently masked in the bootstrap.

The `dq` mechanism, concretely: the driver builds the quote as a string literal `mkStr(#sfromCodes(Cons(34,
Nil)))`; `#coreir.encode` escapes it to `\"` and `run-ir` reads it back as one `"`. So C_min emits a string
literal as `dq ++ content ++ dq` and its own source never contains a `"`-inside-a-string.

Open (optional capstone): the **spike** (the UniML ScalaScript dialect) bootstrapping C_min ÔÇö
`spike(cmin.L) Ôëí ssc1-front(cmin.L)` byte-identical ÔÇö would tie the fixpoint to the p6.0 differential
invariant. The fixpoint above stands on its own with the v2 reference front as the bootstrap.
