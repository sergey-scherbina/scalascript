# ScalaScript 2.1 native product Mirror and custom derives

## Overview

The self-hosted standard frontend must synthesize portable product metadata and
custom typeclass evidence for a case class `derives` clause. The contract is
language data in CoreIR: it must not load Scala reflection, the Scala compiler,
Scalameta, the v1 interpreter's `DerivesRuntime`, or a compatibility frontend.

## Interface

```scalascript
trait Csv[A]:
  def header: String

object Csv:
  def derived(m: Mirror): Csv[Any] = CsvInstance(m.elemLabels.mkString(","))

case class Person(name: String, age: Int) derives Csv

val mirror = summon[Mirror.Of[Person]]
val csv = summon[Csv[Person]]
```

For each top-level product case class, `Mirror.Of[T]` and
`Mirror.ProductOf[T]` resolve to the same immutable portable value. Its
`label`, `elemLabels`, and `elemTypes` fields preserve source declaration order.
The `deriving.Mirror.*` spellings are exact aliases.

For every custom typeclass named by `derives TC`, the standard frontend emits
one immutable `TC[T]` value equal to `TC.derived(mirror)`. The companion method
is invoked once during top-level initialization; later `summon[TC[T]]`
references reuse that value.

## Behavior

- [x] `summon[Mirror.Of[T]]`, `summon[Mirror.ProductOf[T]]`, and their
      `deriving.Mirror` aliases resolve for a declared product case class.
- [x] Mirror `label`, `elemLabels`, and `elemTypes` preserve the case-class
      name, field order, and portable source type spellings.
- [x] `derives TC` invokes the known companion's one-argument `derived` method
      with that Mirror and registers the result as exact `TC[T]` evidence.
- [x] Mirror and derived evidence are ordinary first-class immutable values;
      repeated summon does not reconstruct or re-run derivation.
- [x] A missing or unsupported evidence key remains an exact bounded
      `unbound global: summon` failure; no wildcard, reflection, or fallback is
      introduced.
- [x] `custom-derives-mirror.ssc` prints its exact four compatibility lines on
      installed VM, direct ASM, and reproducible `build-jvm`.
- [x] Full strict parity retires only the custom-derives row, with zero
      mismatch/one-sided errors; release and fresh `v2-*` conformance pass.

## Design

Retain field type spellings and derives names in the self-hosted case-class AST
while leaving the existing constructor/field layout unchanged. The lowerer
adds deterministic private globals for the Mirror value and each derived
dictionary, and extends exact given lookup with those generated keys.

The Mirror is a portable `Mirror/3` CoreIR constructor containing a type label
and two ordinary string lists. The shared v2 runtime already owns exact field
dispatch for that value. A custom dictionary is an ordinary CoreIR call to the
known `TC_derived` companion global; no host-side typeclass or reflection API is
added.

## Decisions

- **Self-hosted AST metadata** — chosen because field type spellings and the
  `derives` clause are source semantics erased by the current case-class node.
  Rejected: infer types from runtime values (too late and incorrect for empty
  products or unused constructors).
- **Exact generated given keys** — chosen to compose with the existing named
  given table and preserve missing-evidence failures. Rejected: a runtime
  `summon` catch-all or suffix matching.
- **Portable CoreIR values** — chosen so VM, direct ASM, and `build-jvm` share
  one representation. Rejected: importing v1 `DerivesRuntime`, Scala
  `deriving.Mirror`, compiler reflection, or per-example provider globals.

## Out of scope

- sum-type/enum Mirrors, `ordinal`, `variants`, or `fromProduct`;
- inline compile-time tuple operations and general implicit search;
- structural built-in Eq/Show/Hash/Order recipes or mixed unknown clauses;
- repairing direct-do, named copy, optics, or SQL blockers.

## Results

`db97ad954` extends the self-hosted case-class AST with ordered field type
spellings and derives names while leaving constructor layout unchanged. The
lowerer emits one portable `Mirror/3` global, exact `Mirror.Of` /
`Mirror.ProductOf` aliases, and a cell-backed custom dictionary initialized in
top-level source order through `TC_derived(mirror)`.

The focused fixture proves all four Mirror aliases, two repeated exact
`summon[Csv[Person]]` references, and exactly one observable derivation call on
both VM and direct ASM. The public example prints `Person`, `name|age`,
`String|Int`, and `name,age` on VM, direct ASM, and deterministic `build-jvm`.
