# ScalaScript 2.1 native structural optics

## Overview

The compiler-free standard frontend must preserve the structural meaning of
`Focus[T](_.path)` and `Prism[Outer, Variant]` without executing a path lambda,
loading the v1 macro/runtime, or relying on PluginBridge. A core-free native
provider then applies the same immutable Lens, Optional, Traversal, and Prism
operations to portable values on VM, direct ASM, and `build-jvm`.

## Interface

```scalascript
val age       = Focus[Person](_.age)                 // Lens
val city      = Focus[Person](_.address.city)        // Lens
val maybeCity = Focus[User](_.profile.some.city)     // Optional
val names     = Focus[Roster](_.members.each.name)   // Traversal
val circle    = Prism[Shape, Circle]                 // Prism
```

Supported path steps are a declared case-class field, `.some` through Option,
and `.each` through List. The runtime surface is:

- Lens: `get`, `getOption`, `set`, `modify`, `andThen`;
- Optional: `getOption`, `set`, `modify`, `andThen`;
- Traversal: `getAll`, `set`, `modify`, `modifyAll`, `andThen`;
- Prism: `getOption`, `reverseGet`, `set`, `modify`.

Missing Optional/Prism targets return `None` and leave `set`/`modify` input
unchanged. Traversal visits and rebuilds list elements in source order.

## Behavior

- [x] A field Lens reads, immutably sets, and modifies one registered portable
      case-class field while retaining every sibling field.
- [x] Nested field paths rebuild each enclosing portable product and compose
      through `andThen` with the same result as one combined path.
- [x] `.some` yields Optional semantics across nested `Some` layers and is a
      no-op on the first `None`.
- [x] `.each` yields Traversal semantics with stable `getAll`, `modify`, and
      set-all order for portable Lists.
- [x] `Prism[Outer, Variant]` matches the exact portable variant tag, modifies
      matching values, and leaves non-matching values unchanged.
- [x] The frontend emits only structural step data; malformed/arbitrary Focus
      lambdas fail explicitly and never execute through reflection or fallback.
- [x] The provider has no Scalameta, v1 frontend/interpreter, PluginBridge,
      compiler, Java compiler, or external optics dependency.
- [x] All 23 public `lenses.ssc` lines are exact on standard VM, direct ASM,
      and reproducible `build-jvm`; strict parity retires exactly the last
      language-runtime row and fresh `v2-*` conformance passes.

## Design

### Self-hosted frontend

The parser retains `Focus[...]` and `Prism[...]` type applications as dedicated
AST nodes. Before ordinary selector lowering erases path shape, the lowerer
accepts only a one-parameter lambda whose body is a selector chain rooted at
that parameter. It serializes the chain to portable data:

```text
OField(name) | OSome | OEach
```

`Focus` lowers to `optics.focus(List(steps...))`. `Prism` retains the exact
second nominal type argument and lowers to `optics.prism(variantName)`.
Unsupported shapes lower to a bounded unbound diagnostic sentinel; they do not
fall through to the v1 frontend.

### Core-free provider

`v2/runtime/std/optics-plugin` registers `optics.focus` and `optics.prism`
through `NativePlugin`. The provider uses registered portable field metadata,
Option/List constructors, and `NativePluginContext.invoke` for user callbacks.
It returns method objects containing only immutable structural step lists.

The provider is included in the explicit standard dependency allowlist, root
aggregate, CLI dependency graph, ServiceLoader set, slim/JRE gates, and
compiler-free `build-jvm` closure.

## Decisions

- **Structural extraction before lowering** — chosen because source selector
  chains are deterministic syntax. Rejected: executing the lambda on a proxy,
  bytecode inspection, Scala macros, or reflection.
- **Dedicated core-free provider** — required by the project intrinsic/module
  architecture and keeps optics policy outside the VM kernel. Rejected:
  importing `PluginBridge.registerOptics` into the standard tier.
- **Portable step data ABI** — chosen so VM, direct ASM, and build-jvm consume
  identical CoreIR. Rejected: backend-specific Lens objects or generated host
  source.
- **Bounded `.some`/`.each` surface** — chosen to close the real standard
  example with a truthful contract. Rejected: silently claiming map/index/key
  optics not expressible by the current public Focus syntax.

## Out of scope

- macro-generated arbitrary getter expressions, method calls, filters,
  collection indexes, map keys, or custom optic step extensions;
- changing v1 OpticsRuntime, PluginBridge compatibility behavior, or JS/Rust
  package generators;
- type-level optic law proofs or static distinction between Lens/Optional/
  Traversal in the bounded native checker.
