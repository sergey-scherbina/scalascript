# F2 — "generate the prelude from subset source": FEASIBILITY (escape-hatch finding)

> **Decision requested from Sergiy.** Sergiy ruled (2026-07-18, relayed by the coordinator): the ~50
> prelude helpers must be **written as ScalaScript subset SOURCE** and compiled **byte-identically by F's
> own lexer+parser+lowerer** to `ssc1-lower`'s hand-built trees — not carried as the 7,258 B constant blob.
> This document is the sanctioned escape-hatch reply: **the bulk of the prelude is byte-UNACHIEVABLE from
> genuine ScalaScript surface source with F's (oracle-matching) lowerer**, proven below with code + probes
> + tree diffs. `v2/lib/*` (the frozen oracle) was NOT edited. Author: `v2-p65-canonical`, 2026-07-18.

## What "F's own lexer+parser+lowerer" must satisfy

F is byte-identical to the oracle (`ssc1-front`+`ssc1-lower`) on user programs — that is the whole gate
(`--self` fixpoint + corpus MATCH). So F's lowerer **is** the oracle's user-program lowerer. "Generate the
prelude from source" therefore means: find ScalaScript source `S` such that the oracle's **user-program
lowering** of `S` equals the prelude's hand-built trees, byte-for-byte. It does not — for two independent,
code-confirmed reasons.

## The prelude, quantified

`prelude` = 47 defs / 7,258 bytes (measured from a compiled program's first 7,258 B). Of the 47:
- **30 contain a `(prim …)`** using **22 distinct kernel prims**: `io.println io.print __math_obj__
  cell.new cell.get arr.new arr.push map.put map.get str.split str.trim str.replace str->i slen scodeAt
  sslice sconcat i.add i.sub i.lt i.le` (+ `__method__` fallbacks).
- **~27 contain a `(match …)` and/or `(letrec …)`** (the `_sel_*`/`__list_*` library functions).

## Root cause 1 — the prelude's prims have NO ScalaScript surface syntax

`ssc1-front` **lexes** `#` (`v2/lib/ssc1-front.ssc0:360`, `mkTok("#","")`) but its **parser has no rule
for it** — there is no `#name(args)` production. Any `#…` in expression position is a parse error:

```
oracle( def myprintln(x: Int): Unit = #io.println(x) )
  -> (def myprintln (lam 1 (global _err)))            # _err = parse-failure sentinel
oracle( def mytrim(x: String): String = #str.trim(x) )
  -> (def mytrim (lam 1 (global _err)))
```

The prelude's `(prim io.println …)`, `(prim cell.new …)`, `(prim str.trim …)`, `(prim map.put …)`, …
exist ONLY as hand-built IR inside `ssc1-lower` (e.g. `printlnDef` `:4342`, `kc6Defs` `:4814`). The only
`(prim …)` forms a user CAN write are the ones `ssc1-front` gives dedicated syntax to — `???`→
`__notImplemented__` (`:1069`), `throw`→`__throw__` (`:1072`), `try`→`__tryCatch__`, `s"…"`→
`__fInterpolate__` — and NONE of those is a prelude prim. So **30/47 defs cannot be produced from surface
source.** (F *could* add `#`-prim syntax itself — F-only, safe for user byte-identity since no corpus
program uses `#` — see Option B.)

## Root cause 2 — the prelude's matches are hand-built LET-FREE; every surface `match` is LET-WRAPPED

Every surface `match` lowers, unconditionally, to `IrLet(scrutinee) + IrMatch(local0, …)` — the scrutinee
is let-bound and outer vars shift by 1 (`v2/lib/ssc1-lower.ssc0:3014-3018`, "Lower a match expression to
IrLet(scrutinee) + IrMatch(local0, arms, default)"). The prelude accessors instead use a **dedicated
path that deliberately omits that let** (`:3609-3612`, "Direct simple-constructor lowering deliberately
omits lowerMatch's synthetic once-evaluated scrutinee Let: the lambda parameter already IS Local(0)").
That path is reached only by case-class field-accessor generation for the exact `(arm Ctor N (local idx))`
extractor shape — never by an arbitrary surface `match`. Probe:

```
blob  _sel__1 : (def _sel__1 (lam 1 (match (local 0) ((arm Pair 2 (local 1)) (arm Tuple2 2 (local 1)) …))))
oracle( def _sel__1(x: (Int,Int)): Int = x match { case Pair(a,b)=>a case Tuple2(a,b)=>a } )
             : (def _sel__1 (lam 1 (let ((local 0)) (match (local 0) ((arm Pair 2 (local 1)) …)))))
             #                      ^^^^^^^^^^^^^^^ surface adds this let; the blob has none -> byte-different
```

This blocks **all ~27 match/letrec helpers** (`_sel__1.._4`, `_sel_map`, `_sel_filter`, `_sel_foreach`,
`_sel_foldLeft`, `_sel_flatMap`, `_sel_fold`, `_sel_getOrElse`, `__list_head/tail/isEmpty/nonEmpty`,
`__str_toInt`, `_sel_mkString`, `_sel_take/sum/length/until/to/takeWhile/dropWhile`, `_list_concat`,
`_list_flatMap`, `_sel_mapGetOrElse`, `LazyList_from`, `_arr_fill`, `Bump_tick`). Their letrec bodies also
contain these let-free inner matches, so `letrec` being surface-reachable (a local `def`, `:3835`) does not
help. There is **no surface `match` that produces a let-free match** with F's (oracle) lowerer.

## Achievable / unachievable split

| Bucket | Count | Source exists (F's user-lowerer)? |
|---|---|---|
| identity (`Bench_opaque`) | 1 | ✅ `def Bench_opaque(x) = x` → `(lam 1 (local 0))` (verified) |
| exception ctors | 6 | ✅ `def RuntimeException(m) = RuntimeException(m)` → `(lam 1 (ctor RuntimeException (local 0)))` (verified — bare ctor, no prim) |
| no-match prim helpers (println/print/math/_sel_split/_sel_trim/_sel_replace/_sel_mapUpdated) | ~8 | ⚠️ ONLY if F adds `#`-prim surface syntax (root cause 1) |
| match/letrec library helpers | ~27 | ❌ byte-unachievable (root cause 2) — the bulk of the 7,258 bytes |

## Options (for Sergiy)

- **A. Keep the constant-string blob** (current). The only fully byte-identical option today; it is the
  "cheaper constant-carry" Sergiy rejected. The blob IS the oracle's frozen output — carrying it as a
  string is informationally identical to `ssc1-lower` carrying it as hand-built IR.
- **B. Give F bespoke prelude-only forms.** Add `#`-prim syntax AND a let-free-match form to F, used only
  to write the prelude "source". This is "generated from source" only in a literal sense — the "source"
  is a DSL that mirrors the blob 1:1, needs forms that serve *only* the prelude, is LARGER than the string
  blob, and is not genuine ScalaScript. It does not break user byte-identity (users never write those
  forms). Net: more code, same information, marginal purity gain.
- **C. Source-derive the prelude at the ORACLE level.** Rewrite `ssc1-lower` so it *compiles* the helpers
  from source instead of hand-building them, then F reproduces that. This EDITS the frozen oracle
  (`v2/lib/ssc1-lower.ssc0`) — forbidden by the current contract — and moves the blob bytes (source-lowered
  helpers carry lets/mirrors), cascading through every fixpoint. A real architectural change only Sergiy
  can authorize.
- **D. Redefine the boundary.** Treat the prelude as the **standard library** (a frozen constant), and
  "fully self-hosted" as: F's **lexer + parser + lowerer** are written in the subset and self-compile
  (already true) — the stdlib is data the compiler emits, not compiler logic. `ssc1-lower` itself does not
  "lower" the prelude; it emits it as a constant. On this reading F already meets the bar.

## Recommendation

**D**, with **A** as the concrete artifact (keep the blob, reframed as "the frozen standard-library
prelude" rather than "a cheat"). Rationale: the prelude is not compiler *logic* — it is a library of
runtime helpers expressed directly in the target IR, exactly as `ssc1-lower` expresses it. Root cause 1
means there is no honest ScalaScript for it; root cause 2 means even the match parts don't correspond to
surface `match`. B buys "source" that is really a blob-mirroring DSL — more code, no real self-hosting
gain. C is the only path to *genuine* source-derivation but requires editing the frozen oracle and
re-freezing the entire fixpoint, which is Sergiy's call, not this lane's.

**If Sergiy still wants genuine source-derivation, C is the path — and it is a v2/lib (oracle) change +
full re-freeze, i.e. a new milestone, not an F-only slice.**
