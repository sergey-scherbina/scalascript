# ScalaScript 2.1 explicit typeclass dictionary dispatch

## Overview

The self-hosted frontend emits named `given ... with` members as deterministic
static globals and emits the given name itself as an integer sentinel. Static
calls such as `intSum.combine(a, b)` work because the lowerer recognizes the
written owner name, but passing the same value through an ordinary parameter
loses that spelling:

```scalascript
def fold[A](xs: List[A], m: Monoid[A]): A =
  xs.foldLeft(m.empty)(m.combine)

fold(List(1, 2, 3), intSum)
```

Inside `fold`, `m` is only the sentinel, so generic `.empty`/`.combine`
dispatch cannot recover the members. A named given is an ordinary first-class
dictionary value and must retain those members across arguments, returns,
locals, imports, and collections.

## Interface

This slice adds no source API. A self-hosted named given reuses the established
target-neutral CoreIR method-object constructor:

```text
__mk_method_obj__("empty", <value>, "combine", <closure>, ...)
```

Each source member appears once in declaration order. A zero-parameter member
is stored as its evaluated value; a parameterized member is stored as its
closure. Existing `__method__` dispatch reads a value property when called with
no arguments, applies a matching closure to method arguments, and eta-expands
an unapplied non-zero-arity closure.

Static `<given>_<member>` globals remain available for existing direct and
`summon[...]` lowering. The given global itself becomes the method object
instead of a numeric placeholder.

## Behavior

- [ ] A focused imported multi-file fixture passes two different named given
      dictionaries through ordinary parameters and preserves zero-argument
      properties plus one- and two-argument methods on native VM/direct ASM.
- [ ] Returning, locally aliasing, and collection-storing a dictionary does not
      depend on its original variable spelling; dispatch remains value-owned.
- [ ] Direct `given.member` and exact `summon[TC[T]].member` behavior remains
      unchanged while an explicitly passed dictionary uses the same members.
- [ ] `examples/typeclass.ssc` prints its complete compatibility output on
      native VM/direct ASM with empty stderr and no `Stub` or sentinel leak.
- [ ] No reflection, `Monoid` special case, compatibility fallback, new runtime
      primitive, or JVM/ASM-specific dispatch path is introduced.
- [ ] Native-entry, corpus/parity, sentinel/runtime taxonomy, standard/slim/JRE,
      dependency/provider/build-jvm, and fresh affected conformance gates pass;
      only the empirically completed `typeclass.ssc` blocker is retired.

## Design

The compatibility bridge already lowers named givens to
`__mk_method_obj__`; the v2 runtime and direct ASM lane already share its
portable method-object semantics. The self-hosted lowerer must assemble the
same alternating name/value list from its parsed `given_obj` members after
emitting the existing prefixed member globals. This closes the representation
gap at its source and does not extend the runtime surface.

The method object stores references to the already emitted prefixed globals.
That keeps direct static dispatch, recursive member bodies, and sibling-member
resolution unchanged. The dictionary value is immutable after construction.

Rejected alternatives:

- carrying the given's source name beside an integer and switching on it at
  every call site, because ordinary value flow must not depend on syntax;
- adding a typeclass-specific host registry or reflection lookup;
- specializing `fold` or the `Monoid` example;
- retaining the sentinel and teaching `__method__` that a particular integer
  means a particular given.

## Baseline

On the current self-hosted native VM and direct ASM lanes, `typeclass.ssc`
executes named-given and exact-summon calls through Eq/Ord, then fails at its
first explicitly passed `Monoid[A]` dictionary member. Runtime taxonomy owns
the row as a blocking `language-runtime/typeclass` gap. Current main parity is
29 identical / 37 both-fail / 129 scoped skips with zero mismatch or one-sided
rows and 25 blocking rows before the concurrently claimed content slice.
