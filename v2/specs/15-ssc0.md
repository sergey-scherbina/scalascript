# 15 — `ssc₀` surface and lowering (the seed contract)

> Status: **v1 IMPLEMENTED** (2026-06-26) — `v2/src/Ssc0.scala` (lexer + parser + lower),
> wired into `v2/ssc` (`ssc run`/`ssc compile`). `ssc₀` is the thin, untyped surface the
> front compiles to Core IR. It is `Core IR + names + a little sugar`: just enough to write
> the next layer comfortably (D5), and no more. front = tokenizer → parser → name resolution
> (de Bruijn for locals, names for globals) → desugar → `coreir.encode`. **No type checking.**
> Verified: the worked example below compiles to exactly `conformance/map.coreir`'s `map` def
> (`conformance/check.sh`). Not yet implemented: `import` (single-file only) and bare-`#prim`
> η-expansion — both error with a clear message.

## Design rules

- Everything in `ssc₀` lowers to a `10-core-ir.md` node or a small fixed desugaring. If a
  construct cannot be expressed by lowering, it does not belong in `ssc₀`.
- Untyped. No type annotations, no `given`/`using`, no overloading, no Markdown. Those are
  the *outer* language (`sscc`'s job), not `ssc₀`.
- Strict CBV, matching the kernel. Laziness, if needed in compiler code, is written
  explicitly as a thunk (`() => e` + force), see below.

## Lexical

- **Identifiers** `lower` := `[a-z_][A-Za-z0-9_]*` — variables and `def` names.
- **Constructors** `Upper` := `[A-Z][A-Za-z0-9_]*` — tags (ADT variants, records, tuples).
- **Primitive ref** `#op` := `#` `[a-z][A-Za-z0-9_.]*` — names a Core IR primitive
  (`#i.add`, `#io.print`, `#map.get`, `#coreir.encode`).
- **Literals:** `42` (int), `42n` (big), `4.0` / `4e0` (float), `"…"` (string, same escapes
  as `12-ir-format.md`), `0x…` byte string, `true` `false`, `()` (unit).
- **Comments:** `-- line` and `{- block -}`.
- **Keywords:** `def let rec in if then else match case end import`.

## Grammar (surface)

```
module := item*
item   := "def" lower "=" expr            -- top-level definition (mutually recursive group)
        | "import" string                  -- include another ssc₀ file's defs (flat namespace)

expr   := lambda | letE | ifE | matchE | app
lambda := "(" params? ")" "=>" expr        -- (x, y) => e   ;  () => e is a thunk (Lam 0)
        | lower "=>" expr                   -- x => e        (sugar for (x) => e)
params := lower ("," lower)*
letE   := "let" bind ("," bind)* "in" expr -- let x = e1, y = e2 in body   (sequential let*)
        | "let" "rec" lbind ("," lbind)* "in" expr   -- recursive λ group (LetRec)
bind   := lower "=" expr
lbind  := lower "=" lambda                  -- letrec bindings must be lambdas
ifE    := "if" expr "then" expr "else" expr
matchE := "match" expr "{" arm+ "}"
arm    := "case" pat "=>" expr
pat     := Upper ("(" lower ("," lower)* ")")?   -- shallow: Tag(x,y) | Tag | _
        | "_"                                      -- default
app    := atom (args)*                      -- left-assoc application; f(a)(b), f(a, b)
args   := "(" (expr ("," expr)*)? ")"        -- f()  is a zero-arg call (thunk force)
atom   := lower | Upper args? | "#"op args? | literal | "(" expr ")"
```

Notes:
- **Constructor application** `Tag(a, b)` builds data; bare `Tag` is the nullary constructor.
- **Primitive application** `#op(a, b)`; a bare `#op` (no args) is the primitive wrapped as
  a function value (η-expanded on lowering, see below).
- **Patterns are shallow** (one level, variable sub-binders only). Nested patterns are
  written as nested `match`. This keeps the seed tiny; `sscc` may offer richer patterns in
  the *outer* language and lower them itself.
- **Modules are a flat global namespace.** `import "file"` includes that file's `def`s into
  the shared global scope (no qualified names in v1). The seed is given a set of files and
  forms one mutually-recursive global group.

## Lowering `ssc₀ → Core IR`

`L⟦·⟧` with a scope `Γ` mapping local names to de Bruijn depth. Globals (top-level `def`
names) resolve to `Var(Global name)`; unknown lowercase names are an error.

```
module                    ⇒  Program{ defs = [Def(name, L⟦e⟧) for each def name = e],
                                       entry = <the def named `main`, applied to []>,  -- or explicit
                                       … all defs mutually recursive (global group) }

L⟦ literal ⟧              =  Lit(const)
L⟦ x ⟧  (local)           =  Var(Local (depth of x in Γ))     -- right-to-left: last bound = 0
L⟦ g ⟧  (global def)      =  Var(Global g)
L⟦ (x1..xn) => e ⟧        =  Lam n  L⟦e⟧   with x1..xn pushed (xn at index 0)
L⟦ () => e ⟧              =  Lam 0  L⟦e⟧                       -- thunk
L⟦ f(a1..ak) ⟧            =  App L⟦f⟧ [L⟦a1⟧..L⟦ak⟧]           -- f() ⇒ App L⟦f⟧ []  (force)
L⟦ let x=e1,.. in b ⟧     =  Let [L⟦e1⟧, ..] L⟦b⟧             -- sequential; xi bound for later
L⟦ let rec f=λ,.. in b ⟧  =  LetRec [L⟦λ⟧, ..] L⟦b⟧
L⟦ if c then t else e ⟧   =  If L⟦c⟧ L⟦t⟧ L⟦e⟧
L⟦ Tag(a1..ak) ⟧          =  Ctor "Tag" [L⟦a1⟧..L⟦ak⟧]        -- bare Tag ⇒ Ctor "Tag" []
L⟦ match s { case .. } ⟧  =  Match L⟦s⟧ [arms] default?
   case Tag(x1..xk) => b  ⇒  ("Tag", k, L⟦b⟧)  with x1..xk pushed (xk at index 0)
   case _ => b            ⇒  default = L⟦b⟧
L⟦ #op(a1..ak) ⟧          =  Prim op [L⟦a1⟧..L⟦ak⟧]
L⟦ #op ⟧  (bare)          =  Lam k (Prim op [Local (k-1)..Local 0])   -- η-expand to a value
```

### De Bruijn discipline (the one thing the seed must get exactly right)

Pushing a binder of arity `k` shifts every outer local by `k`; the new binders occupy
indices `k-1 … 0` left-to-right (so the **last** name is `0`), matching
[`10-core-ir.md §1.3`](10-core-ir.md). `match` arms push their fields the same way. This is
the single place de Bruijn bugs hide; the conformance fixtures (`map`) exercise it.

## Worked example

```
def double = (n) => #i.mul(n, 2)

def map = (f, xs) =>
  match xs {
    case Nil       => Nil
    case Cons(h, t) => Cons(f(h), map(f, t))
  }

def main = () => map(double, Cons(1, Cons(2, Cons(3, Nil))))
```

lowers `map` to (xs = `Local 0`, f = `Local 1`; inside the `Cons` arm h = `Local 1`,
t = `Local 0`, f shifts to `Local 3`):

```
(def map (lam 2 (match (local 0)
  ((arm Nil 0 (ctor Nil))
   (arm Cons 2 (ctor Cons (app (local 3) (local 1)) (app (global map) (local 3) (local 0))))))))
```

— identical to the hand-written `map` conformance fixture, which is the point: the seed's
output and the hand-authored Core IR must agree.

## Open / deferred

- **Qualified imports / namespaces** — flat-global is enough to write `sscc`; add
  module-qualified names later if the compiler source gets large.
- **String interpolation, richer literals** — sugar `sscc` can offer in the outer language;
  not needed in `ssc₀`.
- **`main` entry convention** — v1: the `def main` (a `() => …` thunk) is the entry, lowered
  to `entry = App (Global main) []`. Revisit if a different entry shape is wanted.
