# v2.2 P6.0 gate-spike — results & verdict

Status: **GREEN, verdict recorded** (2026-07-13). Gate for `specs/v2.2-self-hosted-dialect.md` §9.
Spike code: `uniml/core/src/test/scala/scalascript/uniml/spike/ScalaSpike.scala` (dialect +
projection) and `ScalaSpikeSpec.scala` (proofs + emission). Run: `cd uniml && sbt
"uniml/testOnly scalascript.uniml.spike.ScalaSpikeSpec"`.

## The gate question

> Does UniML's frame VM (`Open`/`Close`/`Emit`/`Reframe`) express operator precedence tolerably, or
> must we lock decision (B) as "UniML structure + a separate Pratt expression layer"?

## Verdict — YES, tolerable; decision (B) is *refined*, not forced

UniML gives **no precedence for free** (expected — the VM encodes nesting, not binding). But a
separate *parser layer* is **not** needed. The clean pattern is:

1. The dialect's structural pass runs a normal **Pratt parse internally** (a few lines of Scala) to
   the correct expression tree — this is the parsing intelligence UniML never claimed to provide.
2. It then **mechanically serialises** that tree to `VmToken`s with one rule: **a frame opens on the
   first token of its subtree and closes (`Reframe.closeAfter`) on the last.** This is source-order
   and lossless; `Reframe` handles the cases where one token must open several frames (a left-nested
   product like `1 * 2 + 3`) or close several (`… 2 * 3` at expression end).
3. UniML's VM rebuilds exactly that tree as a lossless CST.

So the "Pratt layer" lives *inside the dialect* and its output flows into the *same* UniML CST — we
keep one lossless tree (with trivia/spans/error nodes) for tooling, and precedence is correct. No
second parser, no second tree. `SpikeEmit.walk` (the open-on-first / closeAfter-on-last serialiser)
is ~12 lines and dialect-agnostic; the precedence knowledge is isolated in `SpikeParse` (standard
climbing).

## What was proven

**Gate (CST shape), 6/6 green** — `ScalaSpikeSpec`:
- `1 + 2 * 3` → `add(1, mul(2,3))` (`*` binds tighter);
- `1 * 2 + 3` → `add(mul(1,2), 3)` (left-nested product — the `Reframe` multi-open case);
- `(1 + 2) * 3` → `mul(paren(add(1,2)), 3)` (parens override);
- significant-token source order preserved (lossless up to trivia — see caveat);
- `if x then g(x, 1+2) else 0` → the expected `if`/`call`/arg frames.

**End-to-end (projection → existing lowerer → run-ir), 4/4 green** — projection `SpikeProject`
emits the ssc-v2 `Pair(tag,data)` AST (via the `mk*` builders), fed to the **unchanged**
`v2/lib/ssc1-lower.ssc0` `lowerProg`, encoded to Core IR, executed with `run-ir`:

| toy | expr | run-ir | vs `ssc1-front` | Core IR |
|---|---|---|---|---|
| add-mul | `1 + 2 * 3` | 7 | 7 | **byte-identical** |
| mul-add | `1 * 2 + 3` | 5 | 5 | **byte-identical** |
| paren   | `(1 + 2) * 3` | 9 | 9 | **byte-identical** |
| nested  | `2 * (3 + 4) - 5` | 9 | 9 | **byte-identical** |

The Core IR is **byte-for-byte identical** to what the hand-written `ssc1-front` produces — the
projection is faithful, and §7's "reuse the mature lowerer via projection" is validated.

**Dual-compile (scalac oracle), green** — the same expressions as standard Scala 3
(`SpikeToy.scala`, `object SpikeToy { def … }`) compile under scalac (3.8.4) and evaluate to
`7 5 9 9`, identical to ssc-v2 run-ir. The subset is Scala-valid and semantically agrees on the toy.

## Reproduce the end-to-end diff

`SpikeSpec` emits `<SPIKE_OUT>/*.proj` (projection) and `*.toy.ssc` (source) — default
`SPIKE_OUT=…/scratchpad/p6.0`, override via env. For each toy the harness compared:
- reference: `ssc.jar run v2/bin/ssc1-run.ssc0 <toy>` → Core IR → `run-ir`;
- spike: a 2-line ssc0 driver `import "../lib/ssc1-lower.ssc0"; def main = () =>
  #io.print(#coreir.encode(lowerProg(<proj>)))` (placed in `v2/bin/` so the relative import
  resolves) → Core IR → `run-ir`.
Both the executed value and the Core IR text were diffed. (Driver is transient — not committed.)

## Caveats / deferred to later phases (do NOT block P6.1 on these)

- **Trivia losslessness**: the spike's parser skips whitespace tokens (they are lexed but not put in
  the Node tree), so the CST is lossless only up to trivia. Full trivia attachment is P6.2 polish;
  it does not affect the gate (precedence + source order of significant tokens).
- **Error nodes**: the spike has a placeholder `spike.errExpr` fallback but does not implement the
  total error model — that is P6.1.
- **Grammar breadth**: only the toy subset (arith `+ - *`, call, paren, if, single/param def). P6.2
  grows it to full front coverage.
- **Import resolution gotcha**: nested ssc0 `import "..."` resolve relative to the ENTRY file's dir,
  so the driver must live under `v2/bin/` (not an arbitrary dir) for `../lib/…` to resolve.

## Conclusion

Gate passed. Proceed to **P6.1 (error model + typed holes)**. The architecture is confirmed:
UniML owns the lossless CST + structure; a Pratt pass inside the dialect owns precedence and
serialises into that CST via `Reframe`; the projection reuses the existing `ssc1-lower` (byte-exact).
