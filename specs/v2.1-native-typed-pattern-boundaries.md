# ScalaScript 2.1 — native typed-pattern boundaries

## Overview

The self-hosted 2.1 frontend must keep a type annotation on a match pattern
separate from the arm delimiter and body. The general type scanner deliberately
accepts `=>` inside function types, so reusing it for
`case value: IndentContext => value.currentLevel` consumes the arrow and body.
The empty-arm fallback then invents an expression named `Unit`, which lowers as
an unbound value-level global.

The first slice defined the pattern-specific annotation boundary but then
discarded the annotation and lowered the arm as an unconditional binder. That
is insufficient for ordinary imported code such as
`case context: IndentContext => ...; case _ => ...`: the wildcard branch
overwrites the typed branch, so the layout parser silently selects its default
column. The completed contract retains the outer nominal type head and tests
the runtime portable ADT tag before binding. It does not change general
parameter, result, alias, or ascription type scanning.

## Interface

```scalascript
def currentLevel(context: Any): Int = context match
  case value: IndentContext => value.currentLevel
  case _                    => 0

def active(context: Any): Boolean = context match
  case value: Wrapper[IndentContext] if value.enabled => true
  case _                                               => false
```

At delimiter depth zero, a pattern type ends before either the arm arrow `=>`
or the guard keyword `if`. Brackets and parentheses nested inside the type stay
balanced, so parameterized and function-shaped type arguments are consumed as
one annotation rather than exposed to the expression parser.

The outer nominal head (`IndentContext`, `Wrapper`) survives as a typed-pattern
AST node. At runtime it is checked with the target-neutral CoreIR `__isTag__`
primitive at any constructor arity. Type arguments remain erased after the
balanced boundary is parsed.

## Behavior

- [x] `case value: Type => body` leaves `=> body` for ordinary arm parsing.
- [x] `case value: Type if guard => body` leaves the guard and arrow for
      ordinary guarded-arm parsing.
- [x] Nested `[...]` and `(...)` delimiters inside a pattern type are consumed
      before the depth-zero `if` or `=>` boundary is considered.
- [x] General type annotations still accept function arrows; the fix is scoped
      to match patterns and does not change `skipTypeAnnot`.
- [x] A non-empty typed arm never reaches the empty-arm `uid Unit` fallback or
      emits a value-level `IrGlobal("Unit")`.
- [x] A focused multi-file fixture has identical exact VM/direct-ASM output for
      unguarded and guarded typed binders.
- [x] A typed binder matches only its retained outer nominal ADT tag, binds the
      complete scrutinee on success, and falls through on a different tag.
- [x] Guards run only after the nominal tag succeeds; a failed tag or guard
      continues to the next arm without evaluating the arm body.
- [x] Imported `std/parsing/layout.ssc` selects its `IndentContext` arms rather
      than the wildcard defaults, preserving `withIndent(3)` on the native VM
      and direct ASM lanes.
- [x] `examples/dsl-yaml-like.ssc` prints its complete canonical document and
      queries on both native lanes with empty stderr and no placeholder value.
- [x] Native-entry, corpus/parity/taxonomy, portable, standard, deterministic
      build-jvm, and fresh affected conformance gates remain green. Three
      HTTP-owned packaging gates already broken by the concurrent HTTP-fast
      default swap are isolated as `v21-http-fast-standard-tier-cutover` and
      do not change this language slice's result.

## Out of scope

- JVM reflection, platform class tests, or structural subtype checks. The
  portable contract is the nominal tag carried by ScalaScript ADTs.
- Replacing the empty-arm fallback with a new Unit literal representation.
- Changing function-type syntax or the general type-annotation scanner.
- Repairing the separately tracked JSON `PMapped/2` and imported tuple
  collection match gaps.

## Design

Add `skipPatternTypeAt(depth, toks)` beside the general type scanner. It uses
the same delimiter-depth rules, but at depth zero stops on `=>` and keyword
`if` in addition to the ordinary annotation terminators. A thin
`skipPatternTypeAnnot` wrapper consumes the leading colon. `parseMatchArm` is
the only caller.

Keeping two scanners is intentional: a general type can contain an unwrapped
function arrow, while an arm's depth-zero arrow is grammar punctuation. This
preserves the established function-type contract and avoids a host parser,
reflection, compiler fallback, or synthetic Unit runtime special case.

The scanner also returns the first nominal type token and wraps the original
binder/wildcard pattern as `tpat(basePattern, typeHead)`. Lowering treats typed
arms as ordered arms: `__isTag__(scrutinee, typeHead, -1)` gates the body, the
whole scrutinee is introduced into scope for a binder, and failure recursively
lowers the remaining arms. A guarded typed arm performs the tag test first and
then the guard. Rejected: converting a typed binder into a constructor pattern,
because that would bind constructor fields rather than the whole value and
would require knowing the erased constructor arity in the frontend.
