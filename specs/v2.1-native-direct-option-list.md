# ScalaScript 2.1 native direct syntax for Option and List

## Overview

The self-hosted standard frontend must preserve explicit `direct[Option]` and
`direct[List]` blocks until it can lower their bind statements to portable
CoreIR control flow. A fresh `name = rhs` is a monadic bind, not ordinary
mutable assignment; losing that distinction currently leaves an unbound
`direct` global and makes the public direct-syntax example unusable.

## Interface

```scalascript
val result = direct[Option] {
  x = Some(10)
  y = Some(x + 1)
  Some(y * 2)
}

val pairs = direct[List] {
  x = List(1, 2)
  y = List("a", "b")
  List(x.toString + y)
}
```

For the bounded native contract, every fresh assignment in an explicit block
binds the selected monad. `Option` short-circuits on `None`; `List` preserves
source-order Cartesian expansion. The final expression is already monadic and
is returned unchanged.

`val` remains a pure lexical binding. A name declared by `var` remains mutable,
and later assignment to that exact lexical name is an ordinary cell update,
not a monadic bind. A nested explicit direct block is lowered independently and
may be used as an outer bind RHS.

## Behavior

- [ ] `direct[Option]` binds `Some` values in order and short-circuits on the
      first `None` without evaluating later bind bodies.
- [ ] `direct[List]` produces the stable source-order Cartesian product of all
      fresh binds.
- [ ] Pure `val`, mutable `var`, and later assignment to that mutable name keep
      ordinary lexical/cell semantics inside a direct block.
- [ ] Nested direct blocks lower independently; an inner `None` short-circuits
      the outer Option bind exactly once.
- [ ] Unsupported monads and malformed non-monadic bind values fail with a
      bounded ordinary diagnostic; there is no v1 or runtime fallback.
- [ ] `direct-syntax-demo.ssc` prints the exact 11 compatibility lines on
      installed VM, direct ASM, and reproducible `build-jvm`.
- [ ] Full strict parity retires only the direct-syntax row with zero
      mismatch/one-sided errors; release and fresh `v2-*` conformance pass.

## Design

The self-hosted parser retains `direct[M] { ... }` as a dedicated AST node
instead of erasing the type application and wrapping the block in a nullary
lambda. Before ordinary expression resolution, the lowerer walks the block in
source order and rewrites each fresh bind to the existing portable
`rhs.flatMap(name => rest)` surface.

The transform tracks lexical `var` names while walking. Pure/mutable statements
remain ordinary block statements around the recursively lowered continuation.
Nested direct nodes recurse through the same transform. The resulting AST uses
only existing lambdas, blocks, Option/List constructors, and shared collection
method dispatch, so VM, direct ASM, and `build-jvm` consume identical CoreIR.

## Decisions

- **Frontend desugaring to existing flatMap** — chosen because Option/List
  runtime semantics and effect-aware closure calls already exist. Rejected: a
  new runtime `direct` primitive or importing v1 `DirectAnorm`.
- **Exact lexical var tracking** — chosen to preserve the documented
  `var acc; acc = ...` distinction. Rejected: treating every assignment as a
  bind or guessing from runtime RHS shape.
- **Bounded explicit Option/List surface** — chosen to close the real standard
  blocker without pretending the self-hosted checker has general Monad type
  inference. Rejected: transparent compatibility fallback for other monads.

## Out of scope

- implicit/type-directed direct blocks, Either/Async/custom monads, pure-value
  auto-lift, bind-and-discard typing, or postfix `.!`;
- branch/loop/traverse desugaring beyond expressions already returning Option
  or List helpers;
- repairing named copy, optics, or SQL blockers.
