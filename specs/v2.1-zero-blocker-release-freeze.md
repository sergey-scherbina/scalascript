# ScalaScript 2.1 zero-blocker release freeze

## Overview

TI-8.2d5 converts the zero-blocker runtime taxonomy from a decreasing migration
ceiling into an exact release contract. The standard language/runtime/provider
surface must stay at zero blocking rows, while every remaining VM/ASM
`both-fail` row is an explicitly reviewed optional-provider or tools-backend
surface with stable membership and counts.

## Interface

The exhaustive `scripts/v21-self-hosted-core-release-gate` consumes its
`runtime-taxonomy.tsv` through a dedicated exact freeze gate. The expected
release snapshot is stored as data beside the taxonomy fixtures:

```text
example-contract   0
language-runtime   0
optional-provider  7
standard-provider  0
tools-backend      6
blocker-total      0
total              13
```

The existing manifest remains the exact membership list. Any new, removed,
renamed, reclassified, or newly blocking row requires an explicit reviewed
freeze update; a lower count is not silently accepted as release-equivalent.

## Behavior

- [ ] The exact freeze gate accepts the canonical 13-row runtime taxonomy.
- [ ] It rejects category count drift in either direction, blocker growth,
      total drift, missing metrics, and duplicate metrics.
- [ ] The exhaustive self-hosted-core release gate invokes the freeze gate and
      reports the frozen optional/tools counts in its summary.
- [ ] The 195-row frontend/parity/taxonomy suite remains 194/194 checked, 53
      identical / 13 both-fail / 129 scoped skips, with zero mismatch,
      one-sided errors, language gaps, standard gaps, or blocking rows.
- [ ] Fresh `v2-*` conformance remains green without Scala/Java compiler or
      compatibility fallback in the standard release graph.

## Design

Keep the migration ceiling gate unchanged: it remains useful while a future
reviewed row is actively being retired. Add a second small exact-snapshot gate
over the generated runtime taxonomy report and call it only from the full
release path. A self-test uses synthetic reports to prove that both growth and
shrinkage fail, avoiding a release check that merely restates the happy path.

## Out of scope

- Promoting optional PDF/MCP/NFC/remote graph providers into the standard tier;
- implementing quoted-macro, WASM, x402-client, or MCP tool-source compiler
  surfaces in `StandardMain`;
- changing explicit `ssc-tools --compat-frontend` behavior;
- the TI-8.3 default launcher cutover, which follows this freeze.
