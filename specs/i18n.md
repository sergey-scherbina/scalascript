# spec: std i18n — catalog translation, CLDR plurals, reactive UI binding

Status: implemented
Slug: `std-i18n`

## Motivation

Multilingual apps need three things the language does not yet provide as a
reusable unit:

1. **Catalog translation** with graceful fallback — `locale → (key → text)`,
   falling back to a base locale then the raw key so a missing entry never
   throws.
2. **Correct pluralization.** Slavic languages (ru/uk/pl) have a three-way
   plural (`one`/`few`/`many`): `1 файл / 2 файли / 5 файлів`,
   `1 plik / 2 pliki / 5 plików`. An ad-hoc `if n == 1` is wrong. This is a
   correctness concern that belongs in the standard library, not in every app.
3. **Reactive UI binding** — a single shared locale signal so a language switch
   re-renders the whole SPA instantly, client-side, with no reload.

Driven by the `busi` testbed, which ships an EN/RU/UK/PL UI. busi proved the raw
mechanism works with existing primitives (`Map`, `signal`, `computedSignal`,
`signalText`); this spec lifts the reusable machinery into std so every app gets
it — and gets the plural rules right.

## Non-goals

- Full ICU MessageFormat (select/gender/nested) — only `{n}` count interpolation.
- Fractional plural categories (`1.5`) — integer counts only (the `other`
  category for non-integers is not produced; ru/uk/pl integers are
  one/few/many).
- Locale-aware number/date/money formatting — already covered by
  `std/ui/data.ssc` (`mcol`/`dcol` take a `locale`).
- A bundled translation catalog — apps own their catalogs.

## Module 1 — `std/i18n.ssc` (pure core, package `std.i18n`)

Pure, no UI deps; lowers to interpreter + JS + JVM identically (verified by the
conformance suite).

```
type Catalog = Map[String, Map[String, String]]   // locale -> (key -> text)

// Look up `key` in `locale`; fall back to `base` locale, then the raw key.
def translateIn(catalog: Catalog, locale: String, key: String, base: String): String

// Reduce an Accept-Language header ("uk-UA,uk;q=0.9,en;q=0.5") to a supported
// primary subtag, else `fallback` (not `default` — a JS-backend reserved word).
def negotiateLocale(header: String, supported: List[String], fallback: String): String

// Unicode CLDR plural category for an integer count: "one" | "few" | "many" | "other".
def pluralCategory(locale: String, n: Int): String

// Localized plural form: looks up `keyBase + "." + category`, falling back to
// `.other`, then `.one`, then the raw key.
def pluralForm(catalog: Catalog, locale: String, keyBase: String, n: Int): String

// pluralForm with the count substituted for the literal "{n}".
def plural(catalog: Catalog, locale: String, keyBase: String, n: Int): String
```

### CLDR plural rules (integer `i = |n|`, `m10 = i%10`, `m100 = i%100`)

| locale | one | few | many |
|---|---|---|---|
| en (and default) | i = 1 | — | — (else `other`) |
| ru, uk | m10=1 ∧ m100≠11 | m10∈2..4 ∧ m100∉12..14 | else |
| pl | i = 1 | m10∈2..4 ∧ m100∉12..14 | else |

(en returns `one`/`other`; ru/uk/pl return `one`/`few`/`many` for integers.)

## Module 2 — `std/ui/i18n.ssc` (reactive, package `std.ui.i18n`)

Depends on `std/ui` primitives + `std/i18n`. Provides the shared locale signal
and reactive constructors that re-render on a locale switch.

```
val localeSignal: Signal[String]                 // shared, name "locale", default "en"
def setLocale(loc: String): EventHandler         // for custom controls

def localeText(catalog, key, base): TkNode       // reactive <span>, re-renders on switch
def localeHeading(level, catalog, key, base): TkNode  // reactive <hN>
def localePlural(catalog, keyBase, nSig): TkNode // reactive plural <span> on locale + count

def localeSwitcher(locales: List[String], labels: Map[String, String]): TkNode  // button row
```

`localeSignal` is a module-level `val`, so every screen that imports it shares
the same signal instance — one switch updates all of them.

## Reactive heading node (enabling change)

`HeadingNode` carries a static `String`, so headings could not re-render on a
locale switch. Add a reactive sibling:

- `std/ui/nodes.ssc`: `case class SignalHeadingNode(level: Int, signal: Any) extends TkNode`
- `std/ui/typography.ssc`: `def signalHeading(level: Int, sig: Any): TkNode`
- `std/ui/lower.ssc`: lower to `element("h{level}", <heading style>, [signalText(sig)])`

This reuses the existing `element` + `signalText` externs — **no backend/emitter
change**, so it works on all three backends automatically.

## Interpreter changes (required by the reactive helpers)

The reactive helpers are the first std modules to expose `computedSignal`-based
constructors as **parameterized library functions**. That surfaced two
interpreter limitations (JS/JVM emitters were already correct; both were
INT-only), now fixed:

1. **Closure capture across the interpreter boundary** (`EvalRuntime` +
   `SectionRuntime`). Each imported module runs in its own child `Interpreter`,
   and a `computedSignal` thunk is later invoked by the frontend runtime's
   interpreter — not the one that built it. The "drop an eq-global name, re-read
   it live at call time" capture shortcut was unsound there: a library module's
   own `val` (e.g. `localeSignal`) is not in the invoking interpreter's globals →
   `Undefined`. Fix: capture eq-global **vals** (`interp.valNames`), not just
   non-eq frame-locals; and carry a dependency's val-ness across the import
   boundary when its module globals are dumped into the importer. Reassignable
   **vars** still re-read live (so `var g; () => g; g = 2` stays live) — guarded
   by `ClosureCaptureSoundnessTest`.

2. **Reading a `Signal` via `sig()` on the interpreter** (`CallRuntime`). The
   INT backend had no case for *calling* a `ReactiveSignal` (only JS/JVM wired
   signal reads), so `localeSignal()` inside a thunk failed "Not callable".
   Fix: `sig()` returns the signal's current value (Scala `Signal.apply()`),
   enabling server-side/initial render of reactive UIs. The interpreter renders
   the initial-locale snapshot; live switching remains a JS-backend behavior.

## Verification

- `tests/conformance/std-i18n.ssc` (+ `expected/std-i18n.txt`): pure-core
  assertions run through INT + JS + JVM — `pluralCategory` across
  n ∈ {0,1,2,5,11,21,22,25,101,111} for en/ru/uk/pl, `translateIn` fallback
  chain, `negotiateLocale`, `plural` `{n}` interpolation. Cross-backend identity
  is the CLDR-correctness guarantee.
- `tests/e2e/std-ui-i18n-smoke.sh`: render a demo using `localeSwitcher` +
  `localeText` + `localeHeading` + `localePlural`; assert all four languages'
  text and the switcher buttons appear in the rendered HTML.

## Behavior checklist

- [x] `translateIn` returns locale text; falls back to base; then raw key.
- [x] `negotiateLocale` strips region/q-value; returns default when unsupported.
- [x] `pluralCategory` matches CLDR for en/ru/uk/pl across the teens/ones edge cases.
- [x] `pluralForm` falls back `.<cat>` → `.other` → `.one` → key.
- [x] `plural` substitutes `{n}`.
- [x] `localeSignal` is shared across importing modules; `setLocale` switches it.
- [x] `localeText`/`localeHeading`/`localePlural` re-render on locale switch.
- [x] `localeSwitcher` renders one button per locale, switching `localeSignal`.
- [x] `SignalHeadingNode` lowers identically on all three backends.
