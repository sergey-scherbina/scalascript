# ScalaScript 2.1 native named case-class copy

## Overview

The self-hosted standard frontend must retain named arguments on case-class
`.copy(...)` calls until the portable runtime can rebuild the receiver by its
registered source field order. Erasing labels makes `copy(age = 31)` overwrite
field zero and violates ordinary ScalaScript case-class semantics even though
VM and direct ASM agree on the wrong value.

## Interface

```scalascript
case class Person(name: String, age: Int, city: String)

val alice = Person("Alice", 30, "Boston")
val older = alice.copy(age = 31)
val moved = alice.copy(city = "Paris", name = "Alicia")
val positional = alice.copy("Bob", 40)
```

Named overrides are selected by label, independent of source call order. Every
unmentioned field is copied from the receiver. A positional call retains its
existing declaration-order prefix semantics.

## Behavior

- [x] A single named override updates only the requested field.
- [x] Multiple named overrides may appear out of declaration order; all
      unmentioned fields remain unchanged.
- [x] The receiver and every override expression are evaluated exactly once,
      left-to-right.
- [x] Positional `.copy(...)` keeps declaration-order prefix replacement.
- [x] The behavior is generic over registered portable case-class layouts and
      does not special-case the public lenses example or a field name.
- [x] A focused regression is exact on installed VM, direct ASM, and
      reproducible `build-jvm`; full release and fresh `v2-*` conformance pass.

## Design

The parser already emits `narg(label, value)` nodes. The lowerer preserves
those labels only for the receiver-owned `copy` method. It first binds the
receiver and each override to synthetic lexical locals in source order, then
encodes the pure locals through the existing portable `__method__` copy ABI.
Ordinary calls continue through their current named/default-argument handling,
and purely positional copy calls remain byte-for-byte on the existing path.

The runtime uses `__regfields__(tag, names)` metadata emitted for every native
case class to rebuild the same `DataV` tag in source field order. The lexical
bindings make single, left-to-right evaluation explicit in CoreIR before the
multi-argument primitive dispatch; no temporary host objects or reflection are
needed.

## Decisions

- **Preserve labels at the frontend/runtime seam** — chosen because the parser
  and runtime already own the two halves of the generic contract. Rejected:
  inferring a receiver's static type from local variable flow in the lowerer.
- **Reuse portable copy dispatch** — chosen to keep VM, direct ASM, and
  `build-jvm` on identical CoreIR. Rejected: v1 `OpticsRuntime.evalCopy`, Scala
  reflection, a plugin intrinsic, or transparent compatibility fallback.
- **Keep positional copy unchanged** — chosen because existing callers rely on
  declaration-order prefix replacement. Rejected: forcing every copy call into
  a named-only representation.

## Out of scope

- default-parameter synthesis for constructor calls or mixed positional/named
  copy syntax;
- `Focus`, `Prism`, `Optional`, or `Traversal` lowering;
- changing the compatibility frontend or generated Scala/Java backends.

exact:


The first two rows prove single and out-of-order named overrides; the third
keeps positional prefix behavior; `Box.right` proves the lowering is not tied
to Person/lenses fields; `RCN` proves receiver then overrides evaluate exactly

print the correct two copy rows before the still-explicit `unbound global:
Focus` boundary.
