# ScalaScript 2.1 core-free native Dataset provider

Status: TI-8.2d3g implementation contract (2026-07-11)

## Goal

The standard ScalaScript 2.1 native launcher must execute the local
`Dataset[T]` API without the compatibility frontend, `PluginBridge`, the v1
interpreter, Scalameta, Scala compiler, or Java compiler. The same provider and
portable values serve the VM, direct ASM, and `build-jvm` routes.

This slice closes the three blocking local examples:

- `examples/dataset-stats.ssc`;
- `examples/dataset-word-count.ssc`;
- `examples/dataset-parallel-sum.ssc`.

Spark datasets, distributed MapReduce, and generator construction/conversion
remain separate explicit backend/provider surfaces. The provider must not
pretend those surfaces are local Dataset operations.

## Baseline

Installed `origin/main` through `c22b2d4fa` has no standard native Dataset
provider. VM rejects `Dataset.of`, `Dataset.fromFile`, and `Dataset.fromList` as
unhandled runtime effects. Direct ASM leaks `Op/3`; the 100,000-element
`fromList` payload additionally overflows the diagnostic renderer. Runtime
taxonomy therefore contains three blocking Dataset rows.

## Ownership and representation

The provider lives in `v2/runtime/std/dataset-plugin`, depends only on the v2
native plugin SPI/core graph, and is discovered by `ServiceLoader` in every
standard distribution. It registers the static names `Dataset.of`,
`Dataset.fromList`, and `Dataset.fromFile` directly. No `__fallback__` name is
part of the standard contract.

A Dataset value is an opaque `Value.ForeignV(Value.NamedMethodObj)` containing
an immutable lazy plan:

1. a source thunk;
2. an ordered vector of transformation stages;
3. the selected local/parallel execution mode.

Transformations return a new plan and do not evaluate the source. A terminal
evaluates the complete plan each time, so the value stays reusable and source
or callback failures are observed at the terminal boundary.

## Local API contract

The provider supplies these constructors and operations:

- constructors: `of`, `fromList`, `fromFile`;
- pointwise lazy stages: `map`, `filter`, `flatMap`;
- ordered/global lazy stages: `take`, `drop`, `distinct`, `groupBy`,
  `reduceByKey`, `sortBy`;
- Dataset combinators: `union`, `intersect`, `zip`, `zipWithIndex`;
- execution selectors: `runLocal`, `runParallel`;
- terminals: `collect`, `count`, `reduce`, `fold`, `foreach`, `first`, `min`,
  `max`, `sum`, `avg`, `top`, `takeOrdered`, `countByValue`, `partition`,
  `mkString`, `toMap`, `toSet`, and `saveToFile`.

`fromGenerator` and `toGenerator` are owned by the independently queued
generator provider. Until that provider is installed they fail explicitly;
they never manufacture a stub or compatibility fallback.

## Determinism and values

- Source and transformation order is preserved.
- `union` preserves both sides and duplicates. `intersect` preserves the first
  occurrence order of distinct left values present on the right. `zip` stops at
  the shorter side.
- Group keys and `countByValue` entries are emitted in first-seen order.
- `sortBy`, `top`, and `takeOrdered` are stable for equal keys/values.
- Numeric comparison accepts portable Int/Long, BigInt, Float/Double, and
  Decimal values; string and boolean ordering is deterministic. Unsupported
  mixed values use the portable display ordering rather than JVM object order.
- `count` and frequency values are portable `IntV` longs. `avg` is a portable
  floating value. Empty `min`, `max`, `avg`, and `reduce` fail with a bounded
  Dataset-specific error.
- Lists are converted iteratively in both directions. A 100,000-element
  `fromList`/`collect` path must not recurse through the host stack.
- File operations use UTF-8 and the standard JVM filesystem only. They add no
  compiler or parser dependency.

## Parallel mode

`runParallel` uses JDK 21 virtual threads for `map`, `filter`, and `flatMap`
stages when more than one input chunk exists. Results are merged by chunk index,
so values have exactly the same order as `runLocal`. Global stages execute over
the ordered merged sequence. Callback side effects are not ordered and are not
part of Dataset's deterministic value contract.

`runLocal` always evaluates sequentially. Either selector returns a new Dataset;
it does not evaluate immediately.

## Failure and isolation rules

- A callback with wrong arity or wrong result shape fails at the owning Dataset
  method with a bounded message.
- `filter` requires Boolean and `flatMap` requires a ScalaScript list.
- A non-Dataset argument to `union`, `intersect`, or `zip` is rejected.
- No missing method becomes `Stub`, `Op`, or an implicit v1 call.
- The standard provider jar must pass strict dependency and class-load gates;
  it may reference the plugin SPI, v2 core, Scala runtime, and JDK only.
- The Scala 3 seed remains the bootstrap authority and is not regenerated from
  a compatibility frontend.
