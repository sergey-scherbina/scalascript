# x402 — HTTP Payment Protocol

x402 uses HTTP 402 for machine-to-machine micropayments. The server returns
`402 Payment Required` with payment requirements; the client signs an
authorization and retries with an `X-Payment` header.

## Protocol flow

```
Client                         Server                      Facilitator
  │                               │                              │
  │── GET /api/premium ──────────►│                              │
  │◄─ 402 + requirements ─────────│                              │
  │                               │                              │
  │  (sign EIP-712 / CIP-8)       │                              │
  │                               │                              │
  │── GET /api/premium ──────────►│                              │
  │   X-Payment: <proof>          │── verify(proof, req) ───────►│
  │                               │◄─ Ok ────────────────────────│
  │                               │── settle(proof, req) ───────►│
  │◄─ 200 + data ─────────────────│◄─ Ok(txHash) ────────────────│
```

## Payment schemes

```scalascript
enum PaymentScheme:
  // EVM: single exact payment via EIP-3009 transferWithAuthorization
  case Exact(amount: BigInt)

  // EVM: metered billing — client pre-authorizes a budget,
  // server charges per unit (request / token / second)
  case Stream(
    ratePerUnit: BigInt,
    unitName:    String,    // "request", "token", "second"
    maxUnits:    Int,
    maxAmount:   BigInt,
  )

  // Cardano: CIP-8 signed payment proof
  case CardanoExact(
    lovelace: BigInt,
    asset:    Option[CardanoAsset],   // None = ADA
  )
```

## Core types

```scalascript
case class PaymentRequirements(
  scheme:            PaymentScheme,
  network:           Network,
  asset:             Asset,
  payTo:             String,
  resource:          String,
  description:       String,
  maxTimeoutSeconds: Int = 300,
)

case class TransferAuthorization(   // EVM only
  from:        String,
  to:          String,
  value:       BigInt,
  validAfter:  BigInt,
  validBefore: BigInt,
  nonce:       Bytes32,
)

case class PaymentPayload(
  x402Version:   Int = 1,
  scheme:        PaymentScheme,
  network:       Network,
  authorization: TransferAuthorization,     // EVM
  signature:     String,
)

// Cardano payment proof (instead of PaymentPayload for CardanoExact)
case class CardanoPaymentProof(
  address:   String,      // bech32 payer address
  signature: String,      // CIP-8 COSE_Sign1 hex over PaymentRequirements JSON
  key:       String,      // CIP-8 COSE_Key hex
)
```

## Facilitator SPI

```scalascript
enum VerifyResult:
  case Ok
  case Fail(reason: String)

enum SettleResult:
  case Ok(txHash: String)
  case Fail(reason: String)

trait Facilitator:
  def verify(payload: PaymentPayload, req: PaymentRequirements): Async[VerifyResult]
  def settle(payload: PaymentPayload, req: PaymentRequirements): Async[SettleResult]

object Facilitators:
  // Delegates to Coinbase facilitator REST API
  def coinbase(client: CoinbaseClient): Facilitator

  // Self-hosted: direct on-chain verification via EVM JSON-RPC
  def onChainRpc(evm: EvmClient): Facilitator

  // Cardano: CIP-8 signature verification + Blockfrost balance check
  // provider = Blockfrost(apiKey) or Scalus(nodeSocket, ...)
  def cardano(config: CardanoFacilitatorConfig): Facilitator

  // Primary with fallback
  def withFallback(primary: Facilitator, fallback: Facilitator): Facilitator

  // Test mode: always Ok, no real settlement
  def testnet(): Facilitator
```

### Cardano facilitator config

```scalascript
enum CardanoNetwork:
  case Mainnet, Preprod, Preview

enum CardanoProvider:
  // Via Blockfrost REST API
  case Blockfrost(projectId: String, baseUrl: String = "https://cardano-mainnet.blockfrost.io/api/v0")

  // Self-hosted via Scalus + cardano-client-lib (bloxbean)
  // Requires access to a Cardano node or Ogmios endpoint
  case Scalus(
    nodeSocket:  String,                // path to cardano-node unix socket
    ogmiosUrl:   Option[String],        // alternative: WebSocket via Ogmios
    kupoUrl:     Option[String],        // UTxO indexer (optional but recommended)
    signingKey:  Option[String],        // server-side key for settlement
  )

case class CardanoFacilitatorConfig(
  network:         CardanoNetwork,
  provider:        CardanoProvider,
  receiverAddress: String,
)
```

**Settlement difference:**
- EVM: facilitator calls `transferWithAuthorization` on-chain (has signing key)
- Cardano/Blockfrost: client builds + submits Tx; facilitator verifies confirmation
- Cardano/Scalus: facilitator uses `cardano-client-lib` to build and submit Tx server-side

## NonceStore SPI

Prevents double-spend by tracking used nonces.

```scalascript
trait NonceStore:
  def claim(nonce: Bytes32, validBefore: BigInt): Async[Boolean]   // false = already used
  def cleanup(): Async[Unit]

object NonceStore:
  def inMemory(): NonceStore                    // default, single-instance
  def postgres(db: PgClient): NonceStore        // cluster-safe, durable
  def redis(client: RedisClient): NonceStore    // cluster-safe, fast
```

## Settlement modes

```scalascript
trait SettlementQueue:
  def enqueue(payload: PaymentPayload, req: PaymentRequirements): Async[Unit]
  def process(facilitator: Facilitator): Async[Unit]   // drain loop

object SettlementQueue:
  def inMemory(): SettlementQueue
  def kafka(producer: KafkaProducer, topic: String = "x402-settlements"): SettlementQueue
  def postgres(db: PgClient): SettlementQueue

enum SettlementMode:
  // Settle in request body — simple, slightly slower
  case Synchronous

  // Verify in request, settle in background — faster response
  case Async(queue: SettlementQueue)
```

## Server API

```scalascript
case class PaymentConfig(
  requirements:   PaymentRequirements,
  facilitator:    Facilitator,
  nonceStore:     NonceStore     = NonceStore.inMemory(),
  settlementMode: SettlementMode = SettlementMode.Synchronous,
  onSettled:      (PaymentPayload, Request) => Async[Unit] = (_, _) => Async.unit,
)

serve {
  withPayment(
    PaymentConfig(
      requirements = PaymentRequirements(
        scheme      = PaymentScheme.Exact(1_000_000L),   // 1 USDC
        network     = Network.Base,
        asset       = Assets.usdc(Network.Base),
        payTo       = env("WALLET_ADDRESS"),
        description = "Premium API access",
      ),
      facilitator    = Facilitators.coinbase(coinbaseClient),
      nonceStore     = NonceStore.redis(redisClient),
      settlementMode = SettlementMode.Async(SettlementQueue.kafka(kafkaProducer)),
    )
  ) {
    get("/api/premium") { req => Response.json(getData()) }
  }
}
```

## Client API

```scalascript
trait Wallet:
  def address: String
  def signEip712(domain: Eip712Domain, types: Map[String, Any], value: Map[String, Any]): Async[String]

object Wallets:
  def metaMask(network: Network): Future[Wallet]              // browser: window.ethereum
  def privateKey(hex: String, network: Network): Wallet      // Node.js / automation
  def envKey(envVar: String, network: Network): Wallet

val client = Http.client.withX402(
  wallet    = Wallets.envKey("WALLET_KEY", Network.Base),
  maxAmount = 5_000_000L,    // refuse to pay more than 5 USDC per request
)

val data = client.get("https://api.example.com/premium").json[MyData]
```

On Scala.js, `x402ClientJs` provides the browser wallet helper:

```scalascript
val wallet = await(Wallets.metaMask(Network.Base))
val sig = await(wallet.signEip712(domain, types, value))
```

It calls `window.ethereum.request({ method: "eth_requestAccounts" })`,
checks `eth_chainId`, and signs via `eth_signTypedData_v4`. Full
browser `X402Client` retry/interceptor parity is still a follow-up;
the helper gives browser code the same `Wallet` surface for EVM
typed-data signing.
