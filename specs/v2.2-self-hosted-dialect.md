# ScalaScript v2.2 — self-hosted Scala-3 subset dialect on UniML

Status: **DESIGN** (2026-07-13, co-led Sergiy + opus-continue). This is the endgame of the
`uniml-portable` program — its phase 6 (`SPRINT.md:uniml-portable-6-language`), promoted from a
one-line "research spike" to a full design. Spec-first per `/spec-dev`; implementation is phased
and gated on a de-risking spike (§9).

## 1. Overview — the triple invariant

v2.2 is a **self-hosting** compiler for a **minimal Scala-3 subset**, whose parser is a **UniML
dialect**, which lowers to the frozen **ssc0 Core IR** and thence to any backend, and whose source
is simultaneously **valid standard Scala 3**. The novelty is not "a Scala parser on UniML" — it is
collapsing three normally-distinct languages into one:

- **Implementation language** of the v2.2 compiler = the subset.
- **Object language** the compiler parses = the same subset ⇒ it parses *itself* ⇒ self-hosting.
- **Scala 3 ⊇ subset** ⇒ the same source compiles under `scalac` ⇒ a trusted seed *and* a
  differential oracle at every stage.

`ssc0` is the **floor** and does not move: Core IR is frozen (`specs/10-core-ir.md` — 10 value
shapes, 11 nodes; `v2/src/CoreIR.scala`), the VM and backends (run-ir, JS, Rust, JVM/bytecode,
WASM, Swift) already exist. v2.2 lives strictly *above* ssc0 and compiles *down* into it.

Beyond compilation, two tooling properties are **first-class goals**, not afterthoughts:

- **Error resilience** — parse *and* type errors are AST nodes, never exceptions. The pipeline is
  total. A local error becomes a node and does not poison the rest of the tree.
- **Incrementality** — the UniML `Processor[S,I,O]` fold is chunk-incremental; the CST carries
  spans for edit-driven reparse. Formatters, LSP, source-maps, partial evaluation follow from a
  lossless, total pipeline.

- **Mixed syntaxes are the point.** A `.ssc` is a hybrid (YAML front-matter + Markdown structure +
  code blocks). Dialects compose and may nest; a malformed region degrades to an error node and the
  surrounding dialects still parse. "Nothing breaks when syntaxes mix" falls out of UniML's
  containment model, not from special-casing.

## 2. Why UniML earns its place

UniML's VM is a generic tree-assembler with a five-instruction vocabulary — `Open`, `Close`,
`Emit`, `Reframe`, `Report` (`uniml/core/.../TreeVm.scala:5-15`) — that encodes *nesting*, not
precedence or binding. All grammar intelligence lives in a per-dialect *structural pass* (arbitrary
subset-Scala). So UniML replaces the *tree-assembly backend*, not the *parsing intelligence* — the
offside layout, precedence climbing, `given`/`using`, `for`/`match` are exactly as hard either way
(the hand-written `v2/lib/ssc1-front.ssc0`, 2899 lines, is the existing proof of that difficulty).

UniML is nonetheless the right substrate here, for reasons specific to ScalaScript:

1. **Unifies the hybrid.** UniML already parses the *outer* `.ssc` structure losslessly — YAML
   front-matter and Markdown (headings/links/fenced blocks) are done dialects
   (`project_uniml_markdown_0713`; `df213209c` immutable MD engine). The only missing piece is a
   sub-dialect for the *content* of code blocks. Then the **whole file is one lossless CST**.
2. **Error-as-node is native.** `Report` is a first-class VM instruction and the CST is lossless by
   construction — the model is total by design, whereas the hand-written front bails/throws.
3. **Dialect boundaries are natural resync points.** A fence, a dedent, a closed delimiter already
   bound a region; a local error inside it cannot leak out. Mixed-syntax containment is free.
4. **One tree for many languages** ⇒ one formatter/LSP/source-map/incremental engine across YAML,
   Markdown, and the Scala subset. Coherent with design principle #1 ("reuse, don't invent") and #5
   ("each problem keeps its own dialect").

Decision (settled): UniML owns **structure + layout + injection**; a thin **Pratt/precedence
helper** owns expression nesting inside a code block (SPRINT phase 6 asked "whether a grammar/PEG
helper layer is needed" — yes). See §6.

## 3. The bootstrap loop

```
stage 0:  scalac      ⟶  v2.2 compiler (JVM)            trusted seed
stage 1:  stage-0     ⟶  v2.2 compiler (any backend)    via UniML-parse + ssc0-lower
stage 2:  stage-1     ⟶  v2.2 compiler                  stage1 == stage2  ⇒  fixed point
```

`scalac` is the oracle at every stage: on **well-formed subset** source, `scalac`-built and
v2.2-built compilers must produce identical observable behavior (differential test against the
conformance corpus). The dual-compile invariant is stated precisely in §4.4.

Self-hosting is proven when the stage-1 compiler recompiles its own source to a stage-2 that is
byte-identical (or behaviorally identical) to stage-1.

## 4. The subset (Scala 3 ∩ ssc-v2)

### 4.1 What is in

The intersection is already measured (`specs/uniml-portable-gapmap.md`). It is, in essence,
**pure-functional Scala 3**:

- immutable `case class` (constructor params) + methods; `sealed trait` / `enum` / ADT with nested
  pattern matching;
- generics, generic traits `[I,O]` + method dispatch (literally the `Processor[S,I,O]` shape);
- `given`/`using` typeclasses, extension methods, higher-order functions;
- local `var` + `while` inside `def` bodies; `String` ops; standard control flow (`if/then/else`,
  `for`-comprehensions, `match`).

### 4.2 What is out (the wall) and the one lift

Out today (`gapmap:38-59`): mutable object fields, plain non-`case` `class`, anonymous
`new Trait:` instances, `new Array[T](n)`, and unreconciled multi-file `package`/`import`.

**One deliberate lift** (decision A, settled): add an **immutable indexed `Array`** to ssc-v2 —
`gapmap:76` calls it "the floor everything else builds on", and a compiler is uncomfortable without
O(1) indexed storage. Everything else stays out; mutable state is threaded functionally
(immutable `Vector`/`Map`, state-passing folds — exactly how UniML's own rewrite works). Any *new*
subset feature must first prove it is semantically identical under `scalac` (§4.4).

### 4.3 The compiler is written purely-functionally

UniML itself is the existence proof that a non-trivial parser fits the pure subset (its immutable
rewrite is ~complete: core, JSON/YAML/XML lexers, and the Markdown engine, `df213209c`). Symbol
tables become immutable maps; passes thread state; no mutable object fields.

### 4.4 Semantic — not just syntactic — intersection

The governing rule: **a feature is in the subset iff `scalac` and ssc-v2 agree on its observable
behavior**, verified by the differential oracle (§3). "Out" therefore means either *unsupported by
ssc-v2* or *cannot be made to agree* — never an arbitrary exclusion. The landmines and their
resolutions:

- **Integers — `Int`, pinned to width-independent usage (option O2).** ssc has a *single* 64-bit
  wrapping integer (`v2/src/CoreIR.scala:11` `CInt(n: Long) // 64-bit wrapping`,
  `v2/src/Runtime.scala:24` `IntV(val n: Long)`), and the front strips the `L` suffix
  (`v2/lib/ssc1-front.ssc0:269`) — so ssc conflates `Int` and `Long`; standard Scala keeps them
  distinct (32/64). Resolution: the subset writes `Int` (ergonomic, matches ssc's native width)
  under the invariant **subset code must not depend on integer bit-width or overflow**. The oracle
  flags any divergence, and the few width-sensitive spots (interner hashes, byte math) go through a
  **fixed-width helper**, not raw `Int` arithmetic. *(Rejected — O1 = write `Long` everywhere: true
  by-the-letter agreement, but `.length`/array-index return `Int` in Scala and the `L` suffixes make
  compiler code ugly for no practical gain once the invariant holds.)*
- **`given`/`using` — in, constrained to unambiguous resolution.** ssc-v2's given dispatch is proven
  (the whole tagless/typeclass conformance cluster is green, `project_v2_typeclass_mono_0713`).
  Divergence is possible only when resolution leans on priority/specificity/tie-breaks; subset code
  keeps exactly one applicable `given` per call site, and the oracle catches any reliance on order.
- **Laziness — out, because ssc-v2 does not implement it** (capability, not fiat). ssc-v2 has lazy
  *lists* (`LazyCons`/`LazyNil` via explicit thunks) and block-thunk sugar, but **no `lazy val`
  (memoized) and no by-name `=> T`** — the kernel is strict CBV (`specs/10-core-ir.md`). Admit it
  later only if added to ssc-v2 *and* verified to match scalac (by-name on a strict kernel is a
  non-trivial thunking exercise). The compiler does not need it — thread state, use explicit thunks.
- **Equality & ordering — pin to structure.** Use structural equality and ordered/explicitly-sorted
  collections in compiler code; never rely on reference identity or `Map`/`Set` iteration order.

**Invariant:** *well-formed subset source ⇒ `scalac`-built and v2.2-built behavior are identical;
malformed source ⇒ v2.2 degrades to error nodes, `scalac` fails.* We do not chase equivalence on
malformed input.

## 5. The CST and the error/hole model (total pipeline)

The pipeline is **total end-to-end** (decision 1, settled — typed holes):

```
source ──UniML──►  lossless CST (with Error nodes)
        ──project──►  AST  (Pair(tag,data), the shape ssc1-lower already consumes)
        ──lower/type──►  ssc0 Core IR (with typed-hole nodes)
        ──backend──►  target
```

- **Parse errors → CST `Error` nodes.** UniML emits `Report(code,msg,severity)` and keeps building;
  the salvaged partial sub-tree is preserved under the error node. Resync happens at the nearest
  **structural boundary** (dialect/fence/offside/closed-delimiter), optionally per-statement inside
  a code block (decision 4). The rest of the file parses normally.
- **Type errors → typed holes in the IR.** A hole is a node carrying its expected/inferred type — a
  generalization of the existing `???`/`__notImplemented__` (`b77862d7f`). The **lowerer and typer
  must never throw**: an unresolved name, a type mismatch, or a missing arm becomes a hole with a
  diagnostic, and lowering continues so the rest of the program still compiles/executes.
- Only the **Scala-subset** dialect is *executable* in v2.2 (decision 4). Other dialects (YAML,
  Markdown, JSON, …) are lossless data that *project to values/AST*; they add no execution
  semantics. An error node in a non-executable dialect is a diagnostic, not a runtime hole.

This is what makes tooling live on broken code: LSP hover/complete, partial evaluation, and format
all operate on a tree that always exists.

## 6. Dialects: composition, injection, registry

Two mechanisms, one model (decision 2, settled — design for (b), ship (a)+interpolator first):

- **(a) Block composition** — the Markdown dialect sees a fenced block info-string
  (```` ```scalascript ````, ```` ```yaml ````, ```` ```sql ````) and delegates the block content to
  the matching sub-dialect. This is ~already how UniML/`.ssc` structure works; v2.2 adds the
  ScalaScript-code sub-dialect as the delegate for code blocks.
- **(b) Injection** — a region *inside* one dialect hosts another dialect's tree at a marked
  injection point: string interpolators (`s"…"`, `f"…"`, `md"…"` already exist in `ssc1-front`;
  generalize to `sql"…"`, `html"…"`, `json"…"`), and (later) attribute-marked regions. Injection is
  the general case of which the existing interpolators are instances.

Both resolve a dialect through a **registry** keyed by name (fence info-string / interpolator prefix
/ injection tag). v2.2 ships a **fixed built-in set** with the registry *hook* present but **not
user-open** (decision 3 — user-declarable/importable dialects are **deferred**; a `.ssc` cannot yet
define its own dialect). Designing the registry now keeps the CST model stable when user dialects
arrive later.

## 7. Projection to the existing lowerer (reuse, don't rewrite)

The UniML CST is a generic string-labeled tree (`kind`/`role`); `ssc1-lower.ssc0` (4651 lines)
consumes a specific `Pair(tag,data)` AST (`mkInf`/`mkApp`/`mkSel`/`mkMatch`/…). v2.2 adds a
**projection**: CST → `Pair(tag,data)`. This lets the **mature lowerer be reused unchanged** while
only the front is swapped — the migration lever in §9. The oracle for the projection is the output
of the current `ssc1-front` on the same corpus.

## 8. Non-goals / deferred (v2.2)

- **User-extensible dialects** — registry hook only; no user-declared grammars (decision 3).
- **Multiple executable dialects** — only the Scala subset compiles/executes; others are data
  (decision 4).
- **Laziness / by-name / mutable object state / plain classes / anonymous instances** — stay out of
  the subset (§4.2); revisit only with a proven scalac-equivalence.
- **Deep incremental typing** — v2.2 targets incremental *parsing* + a total type pass; incremental
  re-typing (fine-grained invalidation) is later.
- **Rewriting `ssc1-lower` into the subset** — v2.2 reuses it via projection (§7). Porting the
  lowerer out of ssc0 into subset-Scala (so the *whole* compiler is dual-compilable) is a
  follow-on, not a v2.2 gate.

## 9. Phasing (spike-first)

Do not rewrite 7550 lines of ssc0 front+lower at once. Vertical slices, each with a `scalac` +
v2.2 dual-build and a diff against `ssc1-front` output:

- **P6.0 — spike (gate).** A UniML sub-dialect parsing the *minimum*: `def f(x: Int): Int = <expr>`,
  `<expr>` over `+ - * ( )` + call + `if/then/else`. Project CST → `Pair(tag,data)` → existing
  `ssc1-lower` → run-ir. Diff against `ssc1-front` on the same inputs; also compile the spike source
  itself with `scalac` (dual-compile on a toy). **Question answered by the spike:** does precedence
  through UniML come out tolerable, or do we lock decision (B) as "UniML structure + Pratt
  expressions"? Do not proceed to P6.1 until the spike is green and the answer is recorded.
- **P6.1 — error model.** CST `Error` nodes + structural-boundary resync; total projection + typed
  holes into the IR (generalize `__notImplemented__`). Conformance: broken inputs produce partial
  IR + diagnostics, not crashes.
- **P6.2 — grow the dialect** to full coverage (layout, full precedence table, `given`/`using`,
  patterns, `for`/`match`, declarations), differential-tested against `ssc1-front` across the whole
  corpus, until it can replace the front.
- **P6.3 — injection + registry.** ✅ *Hybrid composition + registry hook landed* (`unimlScala`,
  `SscCompose`): a whole `.ssc` (YAML front-matter + Markdown prose + fenced code) parses as **one lossless
  UniML tree** by injection — each dialect sees only its own bytes; the fence/front-matter language is
  resolved through a `DialectRegistry` with a **closed built-in set** (ScalaScript/Markdown/YAML/JSON) that
  users may **extend but not override**; an unregistered language stays inert — and the spliced ScalaScript
  subtree projects to Core IR byte-identical to `ssc1-front` (notes §P6.3, §P6.3b). ✅ *String literals +
  interpolation landed* (§P6.3d, §P6.3e): the spike now lexes `"…"`/`"""…"""` and interpolators; `s"…"`
  (literals + `$name` + `${expr}` **re-parsed by the front itself**) and `md"…"` (→ `__mdStrip__`) project
  byte-identically (`interp-var`/`interp-expr`/`interp-md`/`interp-f`). `${…}` injects back into the host and
  `md"…"` is a boundary injection at an inline token — the interpolator side of P6.3, on the same registry
  hook. **P6.3 is complete** — hybrid composition + registry hook + strings + all four interpolators
  (`s`/`f`/`raw`/`md`), byte-identical across a 44-program corpus. Next: **P6.4 — self-host proof.**
- **P6.4 — self-host proof.** ✅ *Landed* (notes §P6.4). A real compiler written entirely in the subset —
  `selfhost-arith` (tokeniser + parser + stack-machine code generator + VM, `+ 3 * 4 5`→23) and
  `selfhost-eval` (a `let`/variable interpreter with de Bruijn scoping, →56) — compiles to Core IR
  byte-identical to `ssc1-front` **and** runs to the correct answer. The differential oracle delivers the
  fixed-point analog: the new front (spike) and the trusted front (`ssc1-front`) agree byte-for-byte on a
  compiler's source. Not the literal `spike-compiles-spike` fixed point (that needs the spike's own front
  ported into the subset — P6.5-adjacent); what P6.4 proves is the prerequisite: the subset hosts a
  compiler and the spike compiles it faithfully. Corpus: 62 byte-identical programs, 0 fail.
- **P6.5 (follow-on, non-gate).** Port `ssc1-lower` — and the spike front — from ssc0/Scala into the
  subset so the whole compiler is dual-compilable end to end (the literal self-compilation fixed point).

## 10. Open questions / risks

- **Precedence-through-UniML ergonomics** — the P6.0 gate. If `Open`/`Reframe` for operator nesting
  is too awkward, the answer is the Pratt helper (already the leaning), projected into the same CST.
- **CST → `Pair(tag,data)` fidelity** — the projection must reproduce every AST shape `ssc1-lower`
  expects; the corpus diff is the guard.
- **`Array` lift semantics** — must match scalac exactly (bounds, default init) without reopening the
  mutable door; likely an immutable persistent-vector-backed `Array` with O(1)-ish indexing.
- **Multi-file `package`/`import`** — v2's `import "path"` vs. Scala packages is still unreconciled
  (`gapmap:82-83`); needed before the compiler's own multi-file source dual-compiles.
- **Incremental identity** — chunk-streaming exists; edit-driven reparse needs node identity/damage
  bounds — designed for, delivered later.
