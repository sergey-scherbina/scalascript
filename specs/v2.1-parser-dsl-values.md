# ScalaScript 2.1 parser DSL value preservation

## Overview

The self-hosted native frontend now parses, checks, and executes the standard
parser-combinator examples on both the v2 VM and direct ASM lanes, but composed
array, object, and layout-parser values are silently replaced by `Stub`.
ScalaScript 2.1 must preserve the ordinary portable values produced by parser
mapping, list composition, tuple fields, folds, and user ADT construction so a
successful parser run has the same observable value as the compatibility lane.

## Interface

This slice adds no public API. It preserves the existing contracts of:

- `Parser.map`, `flatMap`, `many`, `opt`, and layout-aware `runLayout`;
- ordinary `List.map`, `foldLeft`, append, tuples, and user ADT constructors;
- pattern matching and deterministic rendering of those composed values.

`tests/e2e/v21-parser-dsl-smoke.sh` becomes the release contract for exact
`dsl-json-parser.ssc` and `dsl-yaml-like.ssc` output on native VM and direct
ASM, not merely successful exit and byte agreement.

## Design

Start from the first value that becomes `Stub`, before pretty-printing, and
compare its CoreIR/runtime shape with the same scalar and single-element paths.
Because VM and direct ASM currently produce the same placeholder, the first
suspect is shared lowering or portable runtime dispatch rather than bytecode
emission. Repair the smallest generic missing semantic operation and retain the
existing parser evaluator as ordinary `.ssc` code.

After imported extension dispatch and complete local tuple binding are
restored, YAML exposes a second shared frontend loss: typed binders in the
imported layout evaluator retain their token boundary but not their nominal
runtime test. Complete the existing
[`v2.1-native-typed-pattern-boundaries.md`](v2.1-native-typed-pattern-boundaries.md)
contract so `IndentContext` arms win over wildcard defaults; do not encode
layout, JSON, or YAML semantics in the host.

The sibling `native-head-shadow` slice owned `v2/lib/ssc1-lower.ssc0` when this
work began. Its head/tail repair landed first; this slice rebased that result
before changing the lowerer's independent extension, tuple-binding, and typed-
pattern paths.

## Decisions

- **Exact observable parity** — chosen because exit-zero byte parity can
  preserve the same wrong placeholder on both backends. Rejected: treating
  byte-identical `Stub` output as readiness.
- **Generic value semantics** — chosen because JSON and YAML are ordinary
  parser-combinator clients. Rejected: host-side parser/renderer intrinsics or
  example-specific pattern matching.
- **First-loss regression** — chosen to separate parser construction from
  later rendering/query failures. Rejected: changing several list/tuple/render
  operations at once until the examples happen to pass.

## Out of scope

- New JSON or YAML syntax, parser APIs, or host codecs.
- Compatibility-frontend behavior changes.
- The separately queued imported tuple-collection match blocker.
- Changes to a file under a live foreign claim without coordination.

## Baseline

Fresh `installBin` on 2026-07-11 produces empty stderr and byte-identical
VM/direct-ASM output, but JSON ends with:

```text
[1, 2, 3] => [Stub]
{"a": 1, "b": true} => {Stub}
{"name": "Alice", "scores": [95, 87, 91]} => {Stub}
```

YAML prints:

```text
Parsed successfully.
Stub
server.host = Stub
database.name = Stub
database.pool.max = Stub
```

The compatibility frontend prints the complete arrays/objects, the full
13-line YAML document, and query values `localhost`, `myapp`, and `10`.

## Results

Completed in `9d5f13f95`, `0b5d1c69c`, and `d4cc66736`. Imported selected
extensions now lower through the existing `__methodOrExt__` member-first
dispatcher, so a receiver-owned `List.map` wins before the imported
`Parser.map` fallback. Local tuple destructuring evaluates the tuple once and
binds every component through `_sel__N`. Typed match patterns retain their
outer nominal head and lower ordered `__isTag__` tests, including tag-before-
guard fallthrough. No parser, JSON, YAML, VM, or ASM special case was added.

The focused imported value fixture prints
`1:alpha|2:beta|3:gamma`. The layout probe prints exactly:

```text
ParseOk((), host, Position(2))
ParseOk(List(host), , Position(6))
```

in checked-in expected files. Both native VM and direct ASM match those files
