# Imported `PMapped` evaluation

## Overview

ScalaScript's self-hosted native path must evaluate reified parser-combinator
nodes across module imports. In particular, an imported `runParser` evaluator
must match `PMapped(inner, f)`, evaluate `inner`, and apply `f` without any
parser-specific JVM implementation. This spec also records that the previously
reported `match: no arm for PMapped/2` boundary was a stale staged distribution,
not a source-level matcher defect.

## Interface

```scalascript
case class PMapped(inner: Any, f: Any) extends Parser[Any]

def runParser(p: Any, input: String, pos: Position, ctx: Any): Any
```

For a successful inner parser, the observable contract is:

```scalascript
runParser(PMapped(PSucceed(20), x => x + 2), "", Position(0), NoContext)
// ParseOk(22, "", Position(0))
```

The constructor and evaluator may be defined in imported modules. Their module
boundary must not change constructor tag/arity matching or closure invocation.

## Behavior

- [x] A multi-file native fixture imports the evaluator and constructor modules,
      evaluates `PMapped(PSucceed(20), x => x + 2)`, and prints the exact mapped
      value, remaining-input length, and position offset.
- [x] Native VM and direct ASM both exit zero and print exactly `22`, `0`, `0`.
- [x] The JSON and YAML-like parser examples exit zero on both native VM and
      direct ASM, with byte-identical output inside each example.
- [x] No host-side parser constructor special case is added to the frontend,
      VM, or ASM emitter.
- [x] The imported fixture and parser DSL checks run inside the assembled
      native-entry/release path before any independently tracked later failure;
      fresh affected conformance remains green.
- [x] The full corpus/parity/taxonomy baseline reclassifies the two former
      `PMapped/2` failures and records any unrelated mismatch or release-gate
      blocker under its own BUGS/SPRINT owner rather than hiding it.

## Out of scope

- Correcting the JSON and YAML-like examples' later `Stub` values. Those are
  successful parser executions and are independent semantic/lowering gaps.
- Replacing the standard parser-combinator implementation with host code.
- Changing general ADT match order or constructor representation.

## Design

The regression belongs in the assembled native-entry harness because that is
the real distribution path where the stale artifact was observed. A local
fixture module constructs and evaluates `PMapped`; a second module imports the
result function and checks the returned `ParseOk`. Both VM and direct ASM use
the same source and exact expected output.

The parser DSL examples are checked in a focused smoke that asserts successful
exit and VM/ASM byte parity without canonizing their known `Stub` content as
the intended parser-library semantics.

## Decisions

- **Pin the existing portable evaluator** — chosen because a clean assembly at
  `c227b40ee` already produces the correct imported mapping result on both
  lanes. Rejected: adding a host `PMapped` dispatcher (it would duplicate the
  standard library and hide frontend/runtime regressions).
- **Treat the old failure as stale assembly** — chosen because no source commit
  after the extension receiver fix `878474b8d` changed the relevant path, while
  rebuilding the staged distribution removes the failure. Rejected: patching
  the generic matcher without a current reproducer.

## Results

Landed in `5b16df6df` with taxonomy reconciliation in `06a1ae9bb`. The
multi-file fixture prints exactly:

```text
22
0
0
```

on native VM and direct ASM. `v21-parser-dsl-smoke.sh` proves JSON and YAML-like
examples exit 0 with empty stderr and byte-identical VM/ASM output. The
assembled native-entry gate executes the new exact assertion successfully
before reaching the independently tracked obsolete HTTP `useGzip`
feature-unavailable assertion.

The full post-change baseline is 194/194 native frontend/checker success (plus
one non-code document), 39 runtime-success rows, and strict parity 25 identical
/ 1 mismatch / 40 both-fail / 129 scoped skips. Both former `PMapped/2` rows are
now identical and removed from runtime taxonomy. The one mismatch is the
separately tracked `functional.ssc` final `Stub.mkString` dispatch; sentinel
taxonomy remains 68 rows with zero standard gaps. Fresh affected conformance is
11/11.

The remaining JSON/YAML `Stub` placeholders are not treated as success of their
canonical semantics; they are tracked separately as
`v21-native-parser-dsl-stub-values` and TI-8.2d2s3.
