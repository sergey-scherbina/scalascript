# ScalaScript 2.1 core-free YAML

## Overview

Move the established `std.yaml` parse, access, stringify, and fenced-section
surface onto the compiler-free 2.1 JVM route. The implementation must not load
the v1 interpreter, plugin bridge, Scalameta, Scala CLI, or a compiler. It may
reuse `SimpleYaml`, the project's dependency-free Scala implementation of the
documented YAML subset, behind a dedicated native standard provider.

## Interface

- `parseYaml(text)` returns portable `YStr`, `YNum`, `YBool`, `YNull`, `YArr`,
  and `YObj` values;
- `toYaml(value)` emits deterministic block-style YAML with sorted object keys;
- `yamlType`, `yamlStr`, `yamlNum`, `yamlBool`, `yamlArr`, and `yamlGet` preserve
  the existing total accessor defaults;
- a `yaml` or `yml` fence is parsed in document order and exposed as
  `<SectionId>.yaml`, using the existing heading-to-section identifier rule.

## Behavior

- [x] scalars, nested block maps/lists, flow collections, comments, quoted
  strings, and literal/folded blocks convert to portable `ssc.Value` trees;
- [x] malformed input fails with a bounded `YAML parse error` diagnostic and no
  compatibility fallback;
- [x] accessors return the documented value/default for both matching and
  mismatched tags, including missing object keys;
- [x] `toYaml` is deterministic, sorts object keys, and round-trips every
  supported portable YAML value;
- [x] self-hosted heading tracking retains `yaml`/`yml` fences and resolves
  `<SectionId>.yaml` without a v1 `SectionRuntime` object;
- [x] `yaml-parse.ssc` has exact output on native VM/direct ASM and standalone
  `build-jvm`;
- [x] provider ownership, dependency/class-load, no-compiler JRE, slim,
  corpus/parity/taxonomy/portable, and fresh conformance gates pass without a
  mismatch or one-sided row.

## Out of scope

- anchors, aliases, merge keys, tags, multi-document streams, or full YAML 1.2;
- changing the compatibility interpreter, JS, browser, or Rust implementation;
- making the pure self-hosted front-matter parser depend on this provider;
- native lowering for unrelated JSON/TOML/content structured fences.

## Design

Create `v2/runtime/std/yaml-plugin` depending only on the native plugin SPI and
the standalone `scalascript-yaml` module. Convert Java collections returned by
`SimpleYaml` into `DataV` tags, portable `Cons`/`Nil` lists, and `MapV` object
fields. Register the public globals plus one section-construction hook; no
`ForeignV` or v1 value may cross the provider boundary.

Extend the self-hosted root-source scanner to retain headings while it collects
fences. A YAML fence emits a hidden top-level initialization at its document
position. Selector lowering maps the corresponding section identifier's
`.yaml` field to that hidden portable value. Imported modules continue to
contribute declarations only, and ordinary ScalaScript fence order is
unchanged.

## Decisions

- **Dedicated provider with portable ADT values** — keeps runtime YAML separate
  from compiler/front-matter ownership and works unchanged through VM, direct
  ASM, and packaged JARs. Rejected: reusing the v1 YAML plugin, whose value and
  plugin APIs pull the compatibility graph.
- **Reuse dependency-free `SimpleYaml`** — it is project-owned, already defines
  the public subset, and adds no external parser/codec edge. Rejected: SnakeYAML
  or another third-party parser.
- **Preserve section syntax in the self-hosted scanner** — the source contract
  is `<SectionId>.yaml`; changing the public example to call `parseYaml` twice
  would hide a real frontend gap.
